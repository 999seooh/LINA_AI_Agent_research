# Agent: GA 소식지 기반 타사 보험상품 리서치 Agent

## 00. 프로젝트 목적

이 프로젝트는 각 보험사에서 GA 채널 등에 배포한 소식지 PDF를 기반으로, 월별 보험업계 상품 동향과 회사별 출시상품, 상품 개정사항, 상품 특징을 구조화하여 보고용 비교표와 검토 체크포인트를 생성하는 것을 목적으로 한다.

현재 테스트 단계에서는 `input` 폴더에 저장된 GA 소식지 PDF만 분석 대상으로 한다.

분석 대상은 외부 배포자료 및 공개자료에 한정하며, 고객정보·계약정보·청구정보·심사정보·사내 내부자료는 사용하지 않는다.

---

## 01. 분석 대상 자료

- `input` 폴더 내 각 보험사 GA 소식지 PDF
- 공개 보도자료
- 협회 및 감독기관 공개자료

단, 현재 테스트 단계에서는 우선 `input` 폴더 내 GA 소식지 PDF를 중심으로 분석한다.

---

## 02. 주요 분석 관점

1. 월별 보험업계 상품 동향
2. 보험사별 신상품 출시 현황
3. 기존 상품 개정사항
4. 담보 신설·삭제·확대·축소 내용
5. 가입연령, 보험기간, 납입기간 등 가입조건 변화
6. 간편고지, 일반고지, 유병자 상품 관련 특징
7. GA/TM/CM 등 채널별 운영상 참고 포인트
8. 인수 운영 관점에서 참고할 만한 제한 조건
9. 당사 상품/인수기준과 비교 검토가 필요한 항목

---

## 03. 산출물 형식

현재 테스트 단계에서는 `input` 폴더의 GA 소식지 PDF를 기반으로 브라우저에서 확인 가능한 HTML 보고서를 생성한다.

기본 산출물은 다음과 같다.

1. 브라우저 확인용 HTML 보고서: `output/report.html`
2. HTML 보고서에 적용할 CSS 스타일 파일: `output/style.css`
   
필요 시 보조 산출물로 다음 파일을 생성할 수 있다.

1. Markdown 초안: `output/report.md`
2. Excel 초안: `output/report.xlsx`

PDF 파일은 Agent가 직접 생성하거나 Chrome 등을 이용해 자동 변환하지 않는다.

사용자는 `output/report.html`을 브라우저에서 확인한 뒤, 필요한 경우 브라우저의 인쇄 기능을 통해 PDF로 저장한다.

## 05. 운영안 검토 체크포인트

GA 소식지 및 외부 공개자료를 기반으로 상품 운영안 검토 시 다음 항목을 우선 확인한다.

단, 내부 인수기준이나 내부 운영안은 사용하지 않는다. 소식지 PDF 또는 외부 공개자료상 확인 가능한 내용만 정리한다. 확인되지 않는 항목은 “소식지상 확인 불가” 또는 “원문 확인 필요”로 표시한다.

### 05-1. 상품별 플랜 및 플랜별 담보 구성

- 상품별 플랜명
- 플랜별 포함 담보
- 플랜별 담보 구성 차이
- 플랜별 보장 특징

### 05-2. 상품별 플랜/담보별 가입금액

- 플랜별 담보 가입금액
- 담보별 기본 가입금액
- 담보별 선택 가능 가입금액
- 플랜 간 가입금액 차이

### 05-3. 상품별 담보 최대 가입금액

- 담보별 최대 가입 가능 금액
- 연령/플랜/채널별 최대 가입금액 차이
- 소식지상 최대 가입금액 확인 가능 여부

### 05-4. 진단코드 및 질병 분류

- 소식지 또는 공개자료상 언급된 진단명
- KCD/진단코드가 명시된 경우 해당 코드
- 질병군 또는 질병 분류 기준
- 코드가 명시되지 않은 경우 “진단코드 확인 불가”로 표시

### 05-5. 상품별 알릴의무사항

- 상품별 알릴의무사항
- 일반고지/간편고지 여부
- 고지기간
- 주요 질병 관련 질문 항목
- 상품별 고지항목 차이

### 05-6. 알릴사항 기반 인수 관련 참고사항

- 질병 분류별 인수 관련 언급사항
- 특정 질병 또는 병력에 대한 가입 제한 여부
- 부담보, 할증, 거절, 심사 필요 등 소식지상 확인 가능한 표현
- 내부 인수기준은 사용하지 않고 공개자료상 표현만 정리

### 05-7. 사고 분류 및 사고 관련 기준

- 소식지상 언급된 사고 유형
- 상해/질병/교통사고/재해 등 사고 구분
- 사고 관련 담보 또는 보장 조건
- 사고 분류표가 명시되지 않은 경우 “사고 분류표 확인 불가”로 표시

### 05-8. 인수운영 참고포인트

- 상품 운영상 유의할 조건
- 가입 제한 조건
- 연령, 직업, 병력, 기존 가입금액 등 제한 사항
- GA/TM/CM 등 채널별 운영 차이
- 당사 기준과 비교 검토가 필요한 항목
- 내부 기준을 직접 입력하거나 노출하지 않고 “비교 검토 필요” 수준으로 표현

### 05-9. 출처

- 출처 파일명
- 자료 기준월
- 보험사명
- 페이지 또는 섹션 정보가 확인되면 함께 기재
- 외부 링크가 있는 경우 링크 기재

## 06. 운영안 검토용 HTML 표 구조

HTML 보고서에는 다음 표를 포함한다.

| 보험사 | 상품명 | 플랜 | 담보명 | 가입금액 | 최대 가입금액 | 진단코드/질병분류 | 알릴의무사항 | 인수 관련 참고사항 | 사고분류 | 운영 참고포인트 | 출처 |
|---|---|---|---|---|---|---|---|---|---|---|---|

표 작성 원칙은 다음과 같다.

1. 소식지상 확인되지 않는 항목은 빈칸으로 두지 않고 “확인 불가”로 표시한다.
2. 내부자료나 내부 인수기준은 사용하지 않는다.
3. 상품별 비교가 필요한 항목은 “비교 검토 필요”로 표시한다.
4. 출처 파일명을 반드시 기재한다.
5. 원본 PDF는 수정하지 않는다.

---

## 07. 브라우저 확인용 HTML 보고서 작성 기준

HTML 보고서는 CSS 스타일을 적용하여 브라우저에서 바로 확인 가능한 형태로 작성한다.

### 07-1. 파일 구성

- `output/report.html`: 분석 결과가 담긴 브라우저 확인용 HTML 보고서
- `output/style.css`: HTML 보고서에 적용할 스타일 파일

CSS는 별도 산출물이라기보다, HTML 보고서의 가독성과 보고서형 레이아웃을 구성하기 위한 스타일 파일이다.

### 07-2. HTML 보고서 구성

HTML 보고서는 다음 구조를 기본으로 한다.
1. 리포트 제목
2. 분석 기준 및 출처 안내
3. 월별 업계 동향 요약
4. 보험사별 출시/개정 상품 정리
5. 상품별 주요 특징 비교표
6. 운영안 검토 체크포인트
7. 인수운영 참고포인트
8. 출처 기반 리서치 메모

### 07-3. CSS 적용 기준

CSS는 브라우저에서 보고서를 확인할 때 가독성을 높이기 위해 적용한다.
- 흰 배경 기반의 보고서형 레이아웃
- 제목, 섹션, 표 구분이 명확한 구조
- 표는 border, padding, header 강조 적용
- 과도한 색상이나 장식은 사용하지 않음
- 브라우저에서 열었을 때 바로 검토 가능한 형태
- 필요 시 브라우저 인쇄/PDF 저장이 가능하도록 A4 출력 기준 고려
- 인쇄 시 표와 제목이 어색하게 분리되지 않도록 print 스타일 포함

### 07-4. 작성 원칙

1. 분석 결과는 `report.html` 본문에 작성한다.
2. 스타일은 `style.css` 파일로 분리한다.
3. HTML은 브라우저에서 바로 열어 검토 가능한 구조로 작성한다.
4. Agent는 직접 PDF 변환을 수행하지 않는다.
5. PDF 저장은 사용자가 브라우저에서 직접 수행한다.
6. 원본 PDF는 수정하지 않는다.
7. 모든 분석 항목에는 출처 파일명을 기재한다.
8. 확인되지 않은 내용은 “확인 불가” 또는 “원문 확인 필요”로 표시한다.
9. 내부정보나 비공개자료는 HTML 산출물에 포함하지 않는다.
10. 산출물은 공식 보고서가 아닌 참고용 초안으로 작성한다.


## 08. Prohibited Inputs

다음 정보는 입력받거나 활용하지 않는다.

- 고객정보
- 계약정보
- 증권번호
- 청약번호
- 심사정보
- 청구정보
- 사내 내부 문서
- 내부 시스템 화면
- 내부 DB 구조
- 비공개 자료
- 개인정보 또는 민감정보
- 회사 내부 인수기준 원문
- 회사 내부 운영안 원문

사용자가 위 정보를 입력하려는 경우, Agent는 입력 중단을 안내하고 공개자료 기반 분석으로 범위를 재설정한다.

---

## 09. 분석 원칙

1. 소식지 PDF와 외부 공개자료에 근거해서만 정리한다.
2. 출처 파일명 또는 출처 링크를 반드시 남긴다.
3. 확인되지 않은 내용은 추정하지 않는다.
4. 사실과 업무상 시사점을 분리한다.
5. “당사 기준과 비교 필요” 또는 “비교 검토 필요” 수준으로 표현하고 내부 기준을 직접 입력하거나 노출하지 않는다.
6. 산출물은 참고용 초안으로 작성한다.
7. 원본 PDF는 수정하지 않는다.
8. 분석 결과는 별도 `output` 폴더에 저장한다.
9. 현재 테스트 단계에서는 `input` 폴더 내 GA 소식지 PDF만 분석한다.
10. Agent는 직접 PDF 변환을 기본 작업으로 수행하지 않는다.
