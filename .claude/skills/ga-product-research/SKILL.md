# Agent: 타사 보험상품 리서치 Agent

## 00. 프로젝트 목적

이 프로젝트는 각 보험사에서 GA 채널 등에 배포한 소식지, 상품정리 pdf를 기반으로, 월별 보험업계 상품 동향과 회사별 출시상품, 상품 개정사항, 상품 특징을 구조화하여 보고용 비교표와 검토 체크포인트를 생성하는 것을 목적으로 한다.

현재 테스트 단계에서는 `input` 폴더에 저장된 PDF만 분석 대상으로 한다.

분석 대상은 외부 배포자료 및 공개자료에 한정하며, 고객정보·계약정보·청구정보·심사정보·사내 내부자료는 사용하지 않는다.

이 Agent는 GA 소식지의 모든 정보를 빠짐없이 옮기는 것을 목표로 하지 않는다. 분석 목적은 상품 운영안 검토에 필요한 핵심 정보를 선별하여 구조화하는 것이다.

---

## 00-1. 파이프라인 흐름

```
[1단계] /ga-parser
  └─ input/ga-newsletters/*.pdf 읽기
  └─ output/extracted-summary/{보험사명}_{기준월}.md 생성

        │
        ▼

[2단계] /ga-trend-extractor
  └─ extracted-summary 기반 트렌드·체크포인트 추출
  └─ output/uw-insights/trend-checkpoints.md 생성

        │
        ▼

[3단계] /ga-uw-insights
  └─ trend-checkpoints.md + extracted-summary 기반 UW 인사이트 도출
  └─ output/uw-insights/uw-insights.md 생성

        │
        ▼

[4단계] /ga-report-writer
  └─ 모든 output 기반 HTML 보고서 생성
  └─ output/final-report/report.html + style.css 생성

─────────────────────────────────────────────
[독립 실행] /ga-terms-analyzer
  └─ input/terms/*.pdf 약관 분석 (파이프라인과 무관하게 단독 실행 가능)
  └─ output/terms-analysis/{보험사명}_{상품명}_{기준년월}_terms.md 생성
  └─ 결과는 /ga-uw-insights 또는 /ga-report-writer에 보조 입력으로 연계 가능
```

## 01. 분석 대상 자료

- `iga-product-research/input/` 폴더 내 PDF
- 공개 보도자료
- 협회 및 감독기관 공개자료

단, 현재 테스트 단계에서는 우선 `input/ga-newsletters/` 폴더 내 GA 소식지 PDF를 중심으로 분석한다.

---

## 02. 주요 분석 관점

1. 월별 보험업계 상품 동향
2. 보험사별 신상품 출시 현황
3. 기존 상품 개정사항
4. 간편고지, 일반고지, 유병자 상품 관련 특징
5. GA/TM/CM 등 채널별 운영상 참고 포인트
6. 인수 운영 관점에서 참고할 만한 제한 조건
7. 당사 상품/인수기준과 비교 검토가 필요한 항목

---

## 03. 산출물 형식

현재 테스트 단계에서는 `input` 폴더의 GA 소식지 PDF를 기반으로 브라우저에서 확인 가능한 HTML 보고서를 생성한다.

기본 산출물은 다음과 같다.

1. 브라우저 확인용 HTML 보고서: `output/report.html`
2. HTML 보고서에 적용할 CSS 스타일 파일: `output/style.css`

필요 시 보조 산출물로 다음 파일을 생성할 수 있다.

1. Markdown 초안: `output/report.md`
2. Excel 초안: `output/report.xlsx`

PDF 파일은 Agent가 직접 생성하거나 Chrome 등을 이용해 자동 변환하지 않는다.

사용자는 `output/report.html`을 브라우저에서 확인한 뒤, 필요한 경우 브라우저의 인쇄 기능을 통해 PDF로 저장한다.

## 04. 정보 추출 기준

이 Agent는 GA 소식지의 모든 정보를 빠짐없이 옮기는 것을 목표로 하지 않는다.

분석 목적은 상품 운영안 검토에 필요한 핵심 정보를 선별하여 구조화하는 것이다.

따라서 다음 원칙을 따른다.

1. 모든 문장을 요약하지 않는다.
2. 보고용 비교표에 필요한 항목만 추출한다.
3. 상품 운영, 담보 구성, 가입금액, 고지/인수 관련 참고사항 중심으로 정리한다.
4. 단순 홍보 문구, 판매 독려 문구, 중복 설명은 제외한다.
5. 확인되지 않는 내용은 추정하지 않고 “소식지상 확인 불가” 또는 “원문 확인 필요”로 표시한다.
6. 표는 장문 설명보다 핵심 키워드 중심으로 작성한다.
7. 하나의 셀에 너무 많은 내용을 넣지 않는다.
8. 가입조건 전체를 포괄적으로 정리하려고 하지 않는다.
9. 가입조건성 정보가 필요한 경우, 운영 참고포인트 또는 인수 관련 참고사항에 필요한 범위만 간단히 반영한다.

## 05. 기본 산출물 구조

### 05-1. 보험사별 요약

각 보험사에 대해 다음 구조로 작성한다.

```
## [순번]. [보험사명]

**발행 시기:** [기준월 또는 추정]

**주요 내용:**
- [핵심 상품/담보/캠페인 중심으로 3~7개 bullet]
- [보험료 경쟁력, 담보 특징, 플랜 구성 등 운영 참고 포인트 포함]

**특이사항:** [소식지 구성·형식·특이 메시지 등]
```

작성 원칙:
- 홍보 문구·판매 독려 문구는 제외하고 상품 내용 중심으로 기재
- 수치(보험료, 가입금액, 보장금액)가 명시된 경우 그대로 기재
- 소식지에 없는 내용은 추정하지 않음
- 타사 비교 내용이 소식지에 수록된 경우 그대로 반영

### 05-2. 종합 시장 트렌드 및 인사이트

보험사별 요약 이후 다음 표를 작성한다.

| 트렌드 | 주요 내용 |
|---|---|
| [트렌드명] | [해당 트렌드를 보여주는 보험사 및 상품·담보 사례] |

트렌드 항목은 소식지 전체를 관통하는 공통 키워드·경쟁 포인트·신규 담보 흐름 중심으로 도출한다.

트렌드 아래에 **인사이트** 항목을 추가하여 당사 상품 운영안 검토 시 참고할 시사점을 간략히 정리한다.

```
## 인사이트

- [업계 공통 움직임에서 도출한 당사 검토 포인트]
- [비교 검토 필요 수준으로만 표현, 내부 기준 미사용]
```

### 05-3. 보험사 간 비교표 (선택)

보험사별 요약 이후 주요 항목을 기준으로 보험사 간 비교표를 작성할 수 있다.

| 구분 | 삼성화재 | 현대해상 | DB손보 | KB손보 | 메리츠화재 | 한화손보 | 롯데손보 | 흥국화재 |
|---|---|---|---|---|---|---|---|---|
| 신상품/개정상품 |  |  |  |  |  |  |  |  |
| 담보명 |  |  |  |  |  |  |  |  |
| 고지유형 |  |  |  |  |  |  |  |  |
| 인수 관련 특징 |  |  |  |  |  |  |  |  |
| 업무상 참고 포인트 |  |  |  |  |  |  |  |  |
| 출처 |  |  |  |  |  |  |  |  |

## 06. 보험사별 요약 작성 기준

각 보험사 요약은 소식지에서 다음 정보를 우선 확인하여 bullet 형식으로 기재한다.

- 이달의 핵심 캠페인 상품 또는 주력 상품명
- 주요 담보명 및 보장금액 (소식지에 수치가 명시된 경우)
- 보험료 경쟁력 언급 내용 (타사 비교가 수록된 경우 그대로 반영)
- 간편고지·일반고지·유병자 상품 여부
- 신담보 신설·확대 내용
- GA 운영 관련 안내사항 (완전판매, 채널 정책 등)

소식지에 명시되지 않은 항목은 추정하지 않는다. 특이사항은 소식지의 구성 방식이나 눈에 띄는 메시지를 간략히 기재한다.

### 06-1. 진단코드 및 질병 분류

- 소식지 또는 공개자료상 언급된 진단명
- KCD/진단코드가 명시된 경우 해당 코드
- 질병군 또는 질병 분류 기준
- 코드가 명시되지 않은 경우 “진단코드 확인 불가”로 표시

### 06-2. 상품별 알릴의무사항

- 상품별 알릴의무사항
- 일반고지/간편고지 여부
- 고지기간
- 주요 질병 관련 질문 항목
- 상품별 고지항목 차이

### 06-3. 알릴사항 기반 인수 관련 참고사항

- 질병 분류별 인수 관련 언급사항
- 특정 질병 또는 병력에 대한 가입 제한 여부
- 부담보, 할증, 거절, 심사 필요 등 소식지상 확인 가능한 표현
- 내부 인수기준은 사용하지 않고 공개자료상 표현만 정리

### 06-4. 인수운영 참고포인트

- 상품 운영상 유의할 조건
- 가입 제한 조건
- 연령, 직업, 병력, 기존 가입금액 등 제한 사항
- GA/TM/CM 등 채널별 운영 차이
- 당사 기준과 비교 검토가 필요한 항목
- 내부 기준을 직접 입력하거나 노출하지 않고 “비교 검토 필요” 수준으로 표현

### 06-5. 출처

- 출처 파일명
- 자료 기준월
- 보험사명
- 페이지 또는 섹션 정보가 확인되면 함께 기재
- 외부 링크가 있는 경우 링크 기재

## 07. 종합 시장 트렌드 및 인사이트 작성 기준

보험사별 요약 이후 전체 소식지를 관통하는 공통 흐름을 트렌드 표로 정리한다.

트렌드 항목 도출 기준:
- 2개 이상 보험사에서 공통으로 나타나는 상품·담보·캠페인 방향
- 업계 경쟁 구도에서 눈에 띄는 변화 (보험료 경쟁, 신담보 확산 등)
- 이달 소식지에서 특히 강조된 테마 (예: 레켐비, 5세대 실손 대응 등)

인수운영 관점 참고표 (선택):

| 보험사 | 상품명 | 인수 관련 참고사항 | 운영 참고포인트 | 출처 |
|---|---|---|---|---|

인사이트는 트렌드에서 당사 상품 운영안 검토 시 참고할 시사점을 도출한다.
- “비교 검토 필요” 수준으로만 표현
- 내부 인수기준·내부 운영안은 사용하지 않음
- 소식지 내용에 근거한 시사점만 기재

## 08. 브라우저 확인용 HTML 보고서 작성 기준

HTML 보고서는 CSS 스타일을 적용하여 브라우저에서 바로 확인 가능한 형태로 작성한다.

### 08-1. 파일 구성

- `output/report.html`: 분석 결과가 담긴 브라우저 확인용 HTML 보고서
- `output/style.css`: HTML 보고서에 적용할 스타일 파일

CSS는 별도 산출물이라기보다, HTML 보고서의 가독성과 보고서형 레이아웃을 구성하기 위한 스타일 파일이다.

### 08-2. HTML 보고서 구성

HTML 보고서는 다음 구조를 기본으로 한다.

1. 리포트 제목 (기준월, 분석 대상 보험사 목록)
2. 분석 기준 및 출처 안내
3. 보험사별 요약 (각 보험사: 발행 시기 / 주요 내용 bullet / 특이사항)
4. 종합 시장 트렌드 표
5. 인사이트

### 08-3. CSS 적용 기준

CSS는 브라우저에서 보고서를 확인할 때 가독성을 높이기 위해 적용한다.

- 흰 배경 기반의 보고서형 레이아웃
- 제목, 섹션, 표 구분이 명확한 구조
- 표는 border, padding, header 강조 적용
- 과도한 색상이나 장식은 사용하지 않음
- 브라우저에서 열었을 때 바로 검토 가능한 형태
- 필요 시 브라우저 인쇄/PDF 저장이 가능하도록 A4 출력 기준 고려
- 인쇄 시 표와 제목이 어색하게 분리되지 않도록 print 스타일 포함

### 08-4. 작성 원칙

1. 분석 결과는 `report.html` 본문에 작성한다.
2. 스타일은 `style.css` 파일로 분리한다.
3. HTML은 브라우저에서 바로 열어 검토 가능한 구조로 작성한다.
4. Agent는 직접 PDF 변환을 수행하지 않는다.
5. PDF 저장은 사용자가 브라우저에서 직접 수행한다.
6. 원본 PDF는 수정하지 않는다.
7. 모든 분석 항목에는 출처 파일명을 기재한다.
8. 확인되지 않은 내용은 “소식지상 확인 불가” 또는 “원문 확인 필요”로 표시한다.
9. 내부정보나 비공개자료는 HTML 산출물에 포함하지 않는다.
10. 산출물은 공식 보고서가 아닌 참고용 초안으로 작성한다.

---

## 09. Prohibited Inputs

다음 정보는 입력받거나 활용하지 않는다.

- 고객정보
- 계약정보
- 증권번호
- 청약번호
- 심사정보
- 청구정보
- 사내 내부 문서
- 내부 시스템 화면
- 내부 DB 구조
- 비공개 자료
- 개인정보 또는 민감정보
- 회사 내부 인수기준 원문
- 회사 내부 운영안 원문

사용자가 위 정보를 입력하려는 경우, Agent는 입력 중단을 안내하고 공개자료 기반 분석으로 범위를 재설정한다.

---

## 10. 분석 원칙

1. 소식지 PDF와 외부 공개자료에 근거해서만 정리한다.
2. 출처 파일명 또는 출처 링크를 반드시 남긴다.
3. 확인되지 않은 내용은 추정하지 않는다.
4. 사실과 업무상 시사점을 분리한다.
5. “당사 기준과 비교 필요” 또는 “비교 검토 필요” 수준으로 표현하고 내부 기준을 직접 입력하거나 노출하지 않는다.
6. 산출물은 참고용 초안으로 작성한다.
7. 원본 PDF는 수정하지 않는다.
8. 분석 결과는 별도 `output` 폴더에 저장한다.
9. 현재 테스트 단계에서는 `input` 폴더 내 GA 소식지 PDF만 분석한다.
10. Agent는 직접 PDF 변환을 기본 작업으로 수행하지 않는다.
11. 가입조건을 별도 주요 항목으로 장황하게 정리하지 않는다.
12. 소식지의 모든 정보를 담으려 하지 않고, 운영안 검토에 필요한 핵심 정보만 선별한다.
