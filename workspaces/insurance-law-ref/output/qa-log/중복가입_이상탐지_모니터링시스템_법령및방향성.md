# 중복가입 이상탐지 모니터링 시스템 — 법령 근거 및 방향성

> 작성일: 2026-05-28
> 분류: 법령 참조 / 시스템 개발 기획
> 상태: 참고용 초안 (법무·준법 부서 최종 확인 필요)

---

## 1. 배경 및 목적

동시에 여러 보험을 가입하여 보험금을 편취하는 패턴을 사후점검 중심에서 **조기 Alert 체계**로 전환하기 위한 이상탐지 모니터링 시스템 개발 검토.

---

## 2. 관련 법령 근거

### 2-1. 보험사기방지 특별법 (법률 제21066호, 2025.10.1. 시행)

| 조항 | 핵심 내용 | 시스템 설계 적용 포인트 |
|---|---|---|
| **제2조** (정의) | 보험사기행위 = 보험사고의 발생·원인·내용에 관하여 보험자를 기망하여 보험금을 청구하는 행위 | Alert 탐지 대상 행위의 법적 정의 기준 |
| **제3조** (우선 적용) | 보험사기행위 조사·방지·처벌에 관하여 다른 법률에 우선 적용 | 법령 충돌 시 이 법 우선 |
| **제4조** (보고) | 보험회사는 보험사기 의심 합당한 근거 있는 경우 금융위에 보고할 수 있다 | Alert 발동 → 금융위 보고 흐름의 법적 근거 |
| **제5조①** (보호) | 보험사고 조사 과정에서 보험계약자등의 개인정보를 침해하지 아니하도록 노력하여야 한다 | 탐지 과정 개인정보 처리 기준 |
| **제5조②** (지급 제한) | 대통령령으로 정하는 사유 없이 보험사고 조사를 이유로 지급 지체·거절·삭감 불가 | Alert 단계에서 자동 지급 차단 금지 |
| **제5조의3** (자료 요청) | 금융위는 관계 행정기관·보험회사·정보통신서비스 제공자에 자료 제공 요청 가능. 제공 목적 외 사용 금지 | 외부 데이터 연계 근거. 목적 외 사용 불가 |
| **제6조** (고발 의무) | 합당한 근거 있는 경우 수사기관 고발·수사의뢰 의무. 관련 자료 송부 의무 | Alert 최종 처리 단계. 탐지 로그 수사 증거 활용 가능 |
| **제7조** (입원적정성) | 수사기관은 건강보험심사평가원에 입원적정성 심사 의뢰 가능 | 장기·반복 입원 이상탐지 시 연계 경로 |

### 2-2. 보험사기방지 특별법 시행령 (대통령령 제34816호, 2024.8.14. 시행)

| 조항 | 핵심 내용 | 시스템 설계 적용 포인트 |
|---|---|---|
| **제2조** (보고 형식) | 보고 시 포함 항목: 보험회사명 / 행위 발생 일시·장소 / 관련자 / 행위 내용 | Alert 출력 포맷 기준 |
| **제3조** (지급지체 허용 사유) | ① 약관·법률에 근거 / ② 보험사기 의심 근거 있고 금융위 보고 또는 수사의뢰 완료 / ③ 소송·조정 진행 중 / ④ 금융위 지정 사유 | 지급 보류 연동 가능 조건 |
| **제4조** (금감원 위탁) | 금융위는 보고 접수·자료 요청·심의·시정요구를 금감원에 위탁 | 실무 연계 창구는 금감원 |
| **제5조** (민감정보 처리) | 금융위·금감원·건강보험심사평가원의 건강정보·범죄경력·신용정보 처리 허용. 보험회사 직접 처리는 명시 없음 | 보험회사의 건강정보 직접 처리 근거 불명확 → 법무 확인 필요 |

## 3. 이상탐지 시스템 방향성

> 출처: 이다현 기자, "보험사 FDS, '탐지' 넘어 '예측'으로… AI 고도화 어디까지 왔나", S저널, 2026-04-23
> URL: https://www.s-journal.co.kr/news/articleView.html?idxno=40771

### 3-1. 전환 방향: 사후 적발 → 사전 예측

기존 FDS(Fraud Detection System)의 한계:
- 특정 질병 코드·청구 금액·사고 유형 등 **사전 정의된 규칙(Rule)** 기반 탐지
- 조직화·지능화된 보험사기 수법에 대응 어려움

발전 방향:
- **사전 예측 중심**으로 급속히 진화
- 탐지율보다 **신뢰도(정확도)**가 핵심 경쟁력

### 3-2. 탐지 기준 확장

| 탐지 방식 | 내용 |
|---|---|
| **행동 패턴 분석** | 보험 가입 이력, 진료 패턴, 사고 빈도, 청구 시점을 종합 분석 |
| **네트워크 분석** | 병원·설계사·가입자 간 연결 관계 파악, 반복적 청구 패턴 추적 |
| **머신러닝 위험 점수** | 기존 규칙 기반과 ML 위험 점수를 결합하여 청구 위험도 평가 (KB손해보험 사례) |
| **AI 이상징후 탐지** | 보험금 청구 데이터 기반 사전 탐지 (흥국생명 사례) |

### 3-3. 처리 흐름 전환

```
[기존] 청구 접수 → 지급 → 사후 점검(적발)

[개선] 청구 접수 → 실시간 위험도 계산 → 추가 심사 or 지급 보류 결정
```

### 3-4. 역할 확장

FDS 활용 범위 확대:
- 단순 거래 필터 → 언더라이팅·보험금 심사·사후 관리 연계
- **전사적 리스크 관리 도구**로 확장

### 3-5. 주요 과제

| 과제 | 내용 |
|---|---|
| **오탐지(False Positive)** | 정상 고객이 의심 대상으로 분류 → 지급 지연 및 민원 발생 |
| **블랙박스 현상** | 모델 복잡성 증가로 판단 기준 설명 어려움 → 금융소비자보호법상 설명 의무 충돌 가능성 |
| **신뢰도 우선** | 탐지율보다 정확도(신뢰도)가 핵심 |

## 4. 법령 기반 시스템 설계 원칙 (종합)

| 설계 항목 | 요건 | 근거 |
|---|---|---|
| **Alert 탐지 기준** | "합당한 근거" 요건 충족. 오탐지 최소화 설계 | 보험사기방지 특별법 제4조·제6조 |
| **Alert 출력 포맷** | 보고일시·관련자·행위내용 포함 | 시행령 제2조 |
| **지급 보류 연동** | Alert 단계 자동 지급 차단 불가. 금융위 보고 또는 수사의뢰 완료 후 허용 | 특별법 제5조②, 시행령 제3조 |
| **개인정보 처리** | 탐지 목적 명시, 목적 외 사용 금지. 모델 학습 시 가명처리 | 개인정보보호법 제18조·제28조의2 |
| **건강정보 처리** | 보험회사 직접 처리 근거 불명확 → 법무 확인 필요 | 시행령 제5조 |
| **외부 데이터 연계** | 자료 제공 요청권 근거 확보. 제공 목적 외 사용 금지 | 특별법 제5조의3 |
| **탐지 로그 보존** | 수사 증거자료 제출 대비 무결성·보존 기간 설계 | 특별법 제6조② |
| **입원 이상탐지** | 장기·반복 입원 패턴 탐지 시 수사기관 경유 → 건강보험심사평가원 연계 | 특별법 제7조 |
| **오탐지 대응** | 정상 고객 지급 지연 방지 체계 설계. 이의신청 프로세스 포함 | 특별법 제5조, 금소법 설명 의무 |
| **모델 설명 가능성** | 블랙박스 방지. 판단 근거 설명 가능한 구조 설계 | 금융소비자보호법 설명 의무 |
