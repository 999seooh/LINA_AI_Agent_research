## 1. 배경 및 목적

동시에 여러 보험을 가입하여 보험금을 편취하는 패턴(중복가입 보험사기)을 **사후점검 중심에서 조기 Alert 체계**로 전환하기 위한 이상탐지 모니터링 시스템 개발 검토.

**왜 지금인가:**

- 경기침체기에 보험사기 급증 패턴 반복 확인 (금융위기 시 영국 +17%, 미국 플로리다 +21%)
- 국내 2019년 보험사기 적발 8,809억 원 (전년 대비 +10.4%), 적발인원도 동반 급증
- SNS·블로그 구인광고를 통한 일반 계약자 공모 유인 사례 증가
- 기존 FDS는 사전 정의된 규칙(Rule) 기반으로 조직화·지능화된 수법 대응 한계
- 국내외 보험사들이 Rule 기반 → AI/ML 기반 예측 탐지로 전환 중 (교보생명, 현대해상, KB손해보험, 흥국생명)

## 2. 탐지 기술 진화 흐름

```
[1단계] 규칙(Rule) 기반
  └─ 특정 질병코드·청구금액·사고유형 등 사전 정의된 의심징후 기준 적용
  └─ 조사자 수동 추출 중심
  └─ 한계: 조직화·지능화된 수법 대응 어려움

[2단계] 자동화 + SNA
  └─ 자동화된 의심징후 탐지 + 사회관계망분석(SNA) 도입
  └─ 병원·설계사·가입자 간 연결 관계 파악 (IFAS 활용)
  └─ ICIS 76개 FI 지표 기반 정량 평가

[3단계] ML/AI 기반 예측 (현재 진입 단계)
  └─ 머신러닝 위험 점수 + 기존 Rule 결합 (KB손해보험)
  └─ AI 이상징후 사전 탐지 (흥국생명)
  └─ 비지도학습: 정답 레이블 없이 이상 패턴 탐지
  └─ 효과: False Positive(오탐지) 유의하게 감소

[목표] 실시간 위험도 기반 조기 Alert 체계
  └─ 청구 접수 시점에 실시간 위험도 계산
  └─ 추가 심사 or 지급 보류 여부 자동 판단 지원
  └─ 언더라이팅·보험금 심사·사후 관리 전 단계 연계
```

## 3. 국내 데이터 인프라 현황

중복가입 이상탐지에 활용 가능한 외부 집중 기관:

| 기관 | 시스템 | 보유 데이터 | 활용 포인트 |
|---|---|---|---|
| **한국신용정보원** | ICIS (보험신용정보통합조회시스템) | 43개사 보험계약·보험금 지급 정보 (생보·손보·공제 포함) | 타사 중복계약 확인, 보험사기 유의지표 수신 |
| **금융감독원** | IFAS (보험사기인지시스템) | 보험계약·사고 DB, 76개 FI 지표, 공모의심그룹 | SNA 기반 연결 관계 분석, FI 지표 참고 기준 |
| **건강보험심사평가원** | — | 입원적정성 심사 | 장기·반복 입원 탐지 시 수사기관 경유 연계 |

> 타사 계약 정보는 단일 회사 데이터만으로 조직적 사기 탐지에 한계 → **ICIS 연계가 핵심**
> 신용정보법 제32·33조에 따라 신용정보집중기관 경유 방식 확인 필요 (법무 검토)

## 4. 탐지 기법별 설계 기준

### 4-1. 핵심 탐지 기법 조합

| 탐지 기법 | 내용 | 중복가입 적용 포인트 |
|---|---|---|
| **행동 패턴 분석** | 보험 가입 이력, 진료 패턴, 사고 빈도, 청구 시점 종합 분석 | 단기간 다수 보험 가입 후 청구 집중 패턴 탐지 |
| **SNA (사회관계망분석)** | 병원·설계사·가입자 간 연결 관계, 반복 청구 패턴 추적 | 동일 설계사·병원 경유 중복 가입 그룹 식별 |
| **ML 위험 점수** | 규칙 기반 + ML 위험 점수 결합하여 청구 위험도 평가 | 개인별 FI 지표 기반 위험도 점수화 |
| **이상감지 (Anomaly Detection)** | 정상 패턴에서 벗어난 이상징후 자동 탐지 | 가입 직후 청구(사고근접계약), 보장 한도 근접 청구 반복 등 |
| **비지도학습** | 정답 레이블 없이 이상 패턴 탐지 | 신종 수법 등 기존 규칙에 정의되지 않은 패턴 탐지 |

### 4-2. FI 지표 참고 항목 (IFAS 기준)

IFAS의 76개 개인 단위 FI 지표 중 중복가입 탐지 관련 주요 항목:

| 지표 유형 | 예시 항목 |
|---|---|
| 계약 집중도 | 가입 보험회사 수, 단기간 신규 가입 건수 |
| 사고 패턴 | 사고근접계약(가입 직후 사고 발생) 수, 청구 시점 집중도 |
| 관계 지표 | 사고 당 동승자 수(자동차), 동일 병원·설계사 반복 이용 |

## 5. 법령 제약과 기술 설계의 연결

시스템 각 기능이 법령상 어떤 제약을 받는지 연결한 설계 기준표.

| 시스템 기능 | 기술 설계 | 법령 제약 | 근거 |
|---|---|---|---|
| **Alert 발동 기준** | ML 위험 점수 임계값 설정 | "합당한 근거" 요건 충족 필요. 오탐지 최소화 설계 | 보험사기방지 특별법 제4조·제6조 |
| **Alert 출력 포맷** | 탐지 결과 리포트 자동 생성 | 보고일시·관련자·행위내용 포함 항목 명세 | 시행령 제2조 |
| **지급 보류 연동** | 고위험 Alert → 심사 큐 자동 이관 | Alert 단계 자동 지급 차단 불가. 금융위 보고 또는 수사의뢰 완료 후 허용 | 특별법 제5조②, 시행령 제3조 |
| **오탐지 대응** | 이의신청 프로세스 연동 | 정상 고객 지급 지연 방지 체계. 이의신청 프로세스 포함 | 특별법 제5조, 금소법 설명 의무 |
| **모델 학습 (정형)** | 사내 계약·청구 데이터 학습 | 탐지 목적 명시, 목적 외 사용 금지 | 개인정보보호법 제18조 |
| **모델 학습 (비정형)** | SNS·메모 등 비정형 데이터 학습 | 가명처리 후 학습 가능 | 개인정보보호법 제28조의2 |
| **건강정보 처리** | 진료 패턴·입원 이력 분석 | 보험회사 직접 처리 근거 불명확 → **법무 확인 필요** | 시행령 제5조 |
| **타사 데이터 연계** | ICIS 조회를 통한 중복계약 확인 | 신용정보집중기관 경유 의무 여부 확인 필요 | 신용정보법 제32·33조 |
| **외부 데이터 연계** | 공공데이터·정부기관 데이터 수신 | 자료 제공 요청권 근거 확보. 제공 목적 외 사용 금지 | 특별법 제5조의3 |
| **탐지 로그 보존** | 탐지 이력 DB 저장 | 수사 증거자료 제출 대비 무결성·보존 기간 설계 | 특별법 제6조② |
| **입원 이상탐지** | 장기·반복 입원 패턴 탐지 | 수사기관 경유 → 건강보험심사평가원 연계 | 특별법 제7조 |
| **모델 설명 가능성** | XAI(설명 가능 AI) 구조 설계 | 블랙박스 방지. 판단 근거 설명 가능한 구조 | 금융소비자보호법 설명 의무 |

## 6. Alert 처리 흐름 (기술 + 법령 통합)

```
[계약 체결 단계]
  중복계약 체결 확인 (보험업법 제95조의5, ICIS 조회)
        │
        ▼
[청구 접수 단계]
  실시간 위험도 계산 (ML 위험 점수 + Rule 결합)
        │
   ┌────┴────┐
저위험        고위험 → Alert 발동
   │               │
정상 지급     추가 심사 큐 이관 (자동 지급 차단 불가)
                   │
          ┌────────┴────────┐
       내부 심사         Alert 출력 (시행령 제2조 포맷)
          │               보고일시·관련자·행위내용 포함
          │
   합당한 근거 있음
          │
   금감원 보고 완료 (특별법 제4조, 시행령 제4조)
          │
   ┌──────┴──────┐
지급 보류 허용    필요시 수사의뢰 (특별법 제6조)
(시행령 제3조②)   탐지 로그 증거자료 송부
```

## 7. 주요 과제 및 대응 방향

| 과제 | 내용 | 대응 방향 |
|---|---|---|
| **오탐지 (False Positive)** | 정상 고객이 의심 대상으로 분류 → 지급 지연·민원 발생 | AI 도입으로 오탐지 감소 효과 기대. 이의신청 프로세스 병행 설계 |
| **블랙박스 현상** | AI 모델 복잡성 증가로 판단 근거 설명 어려움 | XAI(설명 가능 AI) 구조 채택. 금감원 보고 시 설명 의무 대응 |
| **데이터 품질·부족** | 보험회사별 사기 건 분류 기준 상이, 수기 입력 상황 발생 | 통계 변수 정의 표준화. ICIS 데이터 연계로 보완 |
| **건강정보 처리 한계** | 보험회사 직접 건강정보 처리 근거 불명확 | 법무·준법 부서 검토 후 처리 범위 확정 |
| **AI 거버넌스** | 차별·사생활침해 문제 발생 가능성 | AI 시스템 점검 체계 마련. NAIC AI 지침 참고 |
| **데이터 표준화** | 탐지 시스템 간 데이터 형식 비표준화 | ICIS 공유 데이터 정의·형식 표준 준수 |

## 8. 미확인 사항 — 법무·준법 부서 확인 필요

- [ ] 보험회사의 이상탐지 목적 **건강정보 직접 처리** 가능 여부 (시행령 제5조)
- [ ] Alert 발동 후 **금감원 보고까지의 내부 처리 기한** 기준
- [ ] 타사 계약 정보 수집 시 **신용정보집중기관 경유** 의무 여부 (신용정보법 제32·33조)
- [ ] 오탐지로 지급 지연 발생 시 **금융소비자보호법상 책임** 범위
- [ ] AI 모델 판단 기준의 **금감원 보고 시 설명 의무** 적용 여부
