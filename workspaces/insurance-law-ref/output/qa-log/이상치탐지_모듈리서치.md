## 1. 이상치 탐지 알고리즘 비교표

### 비지도학습 알고리즘

| 알고리즘 | 원리 | 손해율 데이터 적용 방법 | 장점 | 단점 |
|---|---|---|---|---|
| **Isolation Forest** | 랜덤 분기로 이상치를 조기에 고립시키는 트리 앙상블 | 담보별 (빈도, 심도, MLR) 다변량 입력 → 이상 스코어 산출 | 고차원 데이터에 강함, 연산 속도 빠름, 레이블 불필요 | 밀집 군집 내 이상치 탐지 약함, 임계치 설정 기준 불명확 |
| **LOF (Local Outlier Factor)** | 국소 밀도 비교로 주변 대비 희박한 포인트를 이상치로 판단 | 유사 담보군 내 밀도 비교 → 담보 간 상대적 이상도 산출 | 국소 이상 패턴 탐지 우수 | 고차원에서 성능 저하, 신규 데이터 예측 불가 |
| **Autoencoder** | 입력 재구성 오류(Reconstruction Error)가 큰 샘플을 이상치로 판단 | 정상 기간 손해율 패턴 학습 → 재구성 오류 임계치 초과 시 알림 | 복잡한 비선형 패턴 학습 가능 | 학습 데이터 품질 의존, 과적합 위험, 해석 어려움 |
| **DBSCAN** | 밀도 기반 군집화 → 어떤 군집에도 속하지 않는 포인트를 이상치로 분류 | 담보별 손해율 분포 군집화 → 군집 미분류(Noise) 포인트 탐지 | 클러스터 형태 가정 없음, 노이즈 자동 분리 | eps/min_samples 파라미터 민감, 고차원 불안정 |
| **One-Class SVM** | 정상 데이터만으로 결정 경계 학습 → 경계 밖 포인트를 이상치로 분류 | 정상 손해율 구간 학습 → 경계 이탈 시 플래그 | 이상 레이블 없이 적용 가능, 소량 데이터에 상대적 안정 | 커널 선택·파라미터 민감, 대용량 데이터 연산 비용 높음 |

### 시계열 특화 알고리즘

| 알고리즘 | 원리 | 손해율 데이터 적용 방법 | 장점 | 단점 |
|---|---|---|---|---|
| **SARIMA 잔차 분석** | 계절성·추세·자기상관 제거 후 잔차가 정규분포 이탈 시 이상치 판단 | 월별 담보 손해율 시계열 모델링 → 잔차 3σ 초과 시 알림 | 계절성·추세 통제 가능, 해석 직관적 | 선형 가정, 구조적 변화 적응 느림, 담보 수 많을수록 튜닝 부담 |
| **Prophet 예측구간 이탈** | Facebook Prophet으로 미래 구간 예측 → 실제값이 예측 신뢰구간 이탈 시 이상 | 월별 손해율 예측 → 실제 손해율이 80%/95% 예측구간 이탈 시 플래그 | 결측·이벤트 처리 용이, 설정 단순 | 단기 급변 반응 느림, 구간 이탈 기준 임의성 |
| **Matrix Profile** | 시계열 내 유사 패턴을 전수 비교 → 유사 패턴 없는 구간(Discord)을 이상 탐지 | 손해율 시계열에서 과거 패턴과 완전히 다른 구간 탐지 | 레이블 불필요, 계절성 가정 없음, 패턴 기반 직관적 | 윈도우 길이 설정 민감, 연산량 많음, 패턴 이탈에 특화 |

## 2. 보험 손해율 이상치 탐지 특수 고려사항

### 다변량 입력 설계
- 단일 변수(MLR)만 사용 시 원인 파악 불가 → **빈도(건수/계약수), 심도(건당 보험금), MLR, 청구건수, 경과보험료**를 복합 입력으로 구성
- 빈도 이상 vs. 심도 이상을 분리 탐지하면 UW 대응 방향 차별화 가능
- 입력 변수 간 스케일 차이 크므로 StandardScaler 또는 RobustScaler 전처리 필수

### 소량 담보 처리 (데이터 희소성)
- 청구건수 30건 미만 담보는 통계적 이상치 신뢰도 낮음 → **최소 관측 건수 임계치** 설정 후 탐지 제외 또는 별도 경고 레벨 부여
- 유사 담보군 풀링(Pooling) 또는 베이지안 크레딩(Credibility Weighting) 적용 검토
- 소량 담보는 이상치 탐지보다 **추세 모니터링(연속 증가 여부)** 방식이 실용적

### 계절성·대형 사고 이벤트 마스킹
- 태풍·폭설·집중호우 등 자연재해 발생 월은 정상적 손해율 급등 → **이벤트 마스킹 테이블** 관리 필요
- SARIMA·Prophet은 이벤트 변수를 외생 변수로 투입하여 잔차 왜곡 제거 가능
- 대형 사고 이벤트 미마스킹 시 False Positive 급증 위험

### 레이블 부족 문제 (정상/이상 불균형)
- 보험 손해율 데이터는 실제 이상 사례 레이블이 거의 없음 → **비지도학습 우선 적용**
- 과거 UW 개입 이력(담보 중단·재설계 사례)을 소수 레이블로 활용하는 **반지도학습(Semi-supervised)** 보완 가능
- 초기 운영 시 알고리즘 탐지 결과 + UW 담당자 피드백으로 점진적 레이블 축적 권장

## 3. 모델 설명 가능성(XAI) 연계 방법

### SHAP 적용
- Isolation Forest, Autoencoder 등 결과에 **SHAP TreeExplainer / DeepExplainer** 적용
- 이상 스코어를 높인 변수별 기여도를 수치로 산출
- 예시: "청구건수 급증(기여 60%) + 심도 증가(기여 30%)" 형태의 원인 분해 가능
- 담보별 SHAP Summary Plot으로 패턴 가시화

### LIME 적용
- 개별 이상 포인트에 대해 국소 선형 근사 → 해당 시점에서 어떤 변수가 결정적이었는지 설명
- SHAP보다 계산 빠르나 안정성 낮음 → 보조 수단으로 활용

### UW 담당자 친화적 이상 원인 표시 방법
- 알고리즘 출력을 그대로 노출하지 않고 **알림 카드(Alert Card)** 형식으로 변환
  - 예시: "2026년 4월 [입원의료비] 담보 이상 탐지 — 청구건수 전월 대비 +42%, 건당 보험금 정상 범위 내, 원인 변수: 청구빈도"
- 이상 원인을 **빈도 이상 / 심도 이상 / 복합 이상** 3가지로 분류하여 레이블 부착
- 정상 범위 대비 실제값 위치를 **신뢰구간 차트**로 시각화

## 4. False Positive 관리 기법

- **임계치 이중화:** 1차 탐지(느슨한 임계치) → 알림 생성, 2차 확인(엄격한 임계치) → 에스컬레이션으로 2단계 운영
- **앙상블 투표:** Isolation Forest + SARIMA 잔차 + Prophet 이탈 3개 모델 중 2개 이상 동시 탐지 시에만 알림 발송 (과반수 의결 방식)
- **연속 N회 조건:** 단발성 이상은 무시하고 3개월 연속 이상 탐지 시 정식 알림 — 일시적 계절성 효과·이벤트 스파이크 자연 필터링
- **이벤트 마스킹 연동:** 자연재해·규정 변경 이벤트 발생 월은 자동 억제(Suppression) 처리
- **Human-in-the-loop 구조:** UW 담당자가 탐지 결과에 "정상 확인 / 이상 확인 / 보류" 피드백 입력 → 피드백 데이터로 임계치 자동 재보정 (Active Learning 루프)
- **담보별 개별 임계치:** 소량 담보는 임계치를 완화하고 대형 담보는 엄격하게 설정하는 담보 등급별 차등 임계치 테이블 관리

## 권장 시스템 아키텍처 (요약)

```
[입력 레이어]
  담보별 월별 (빈도, 심도, MLR, 청구건수, 경과보험료)
  전처리: RobustScaler, 이벤트 마스킹 플래그

        ↓

[1차 탐지: 시계열 기반 - 설명 가능]
  SARIMA 잔차 분석 / Prophet 예측구간 이탈
  → 잔차 3σ 초과 또는 예측구간 이탈 시 1차 플래그

        ↓

[2차 탐지: 다변량 ML - 정밀 분석]
  Isolation Forest + LOF 앙상블
  → 2개 모델 중 1개 이상 이상 스코어 임계치 초과 시 2차 플래그

        ↓

[앙상블 의결]
  1차 + 2차 동시 탐지 시 → 정식 알림 발송
  (단발 탐지는 주의 등급, 연속 3개월 시 경보 등급 상향)

        ↓

[XAI 레이어]
  SHAP 기반 원인 변수 기여도 산출
  → Alert Card 형식으로 UW 담당자에게 전달

        ↓

[Human-in-the-loop]
  UW 담당자 피드백 (정상/이상/보류)
  → 피드백 DB 적재 → 임계치 재보정 (Active Learning)
```
