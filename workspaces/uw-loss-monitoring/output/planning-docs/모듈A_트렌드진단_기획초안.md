## 1. 목적

담보별 손해율(MLR) 추이를 정기적으로 진단하여 요율 부족 신호를 조기에 탐지하고, UW 담당자가 적시에 개입할 수 있는 **자동화된 트렌드 모니터링 체계**를 수립한다.

**현행 문제:**
- 담보별 손해율 악화를 정기 보고 시점(분기/반기)에 사후 인지
- 분석 기법·경보 기준이 담당자별로 상이하여 일관성 부족
- 소형·신규 담보에 대한 별도 기준 없이 동일 임계치 적용 → False Alarm 과다

**기대효과:**
- 손해율 이상 신호 조기 감지 → UW 대응(가입 제한·보험료 조정) 선제 실행
- 담보 규모별 차등 기준 적용 → False Alarm 감소
- 경보 이력 축적 → 임계치 자동 재보정 기반 마련

---

## 2. 분석 대상 및 집계 기준

| 항목 | 기준 |
|---|---|
| **분석 단위** | 담보별 × 월별 |
| **기준일** | 매월 말일 (경과보험료 마감 기준) |
| **집계 지표** | MLR, 빈도(Frequency), 심도(Severity), 경과보험료, 청구건수 |
| **최소 분석 기간** | 24개월 이상 (STL 계절분해 적용 요건) |
| **담보 분류** | 대형 / 중형 / 소형·신규 (경과보험료 기준 구간 설정 — 내부 기준 확인 필요) |
| **제외 조건** | 청구건수 30건 미만 담보 → 통계 경보 제외, 별도 정성 리뷰 대상으로 분류 |

---

## 3. 핵심 지표 정의

| 지표 | 산식 | 모니터링 활용 |
|---|---|---|
| **MLR** | 발생손해액 ÷ 경과보험료 × 100 | 담보별 월별 추이 타점 |
| **ELR** | 요율 산출 시 가정 손해율 | MLR/ELR 비율로 요율 적정성 진단 |
| **MLR/ELR 비율** | MLR ÷ ELR | 1.05↑ 주의 / 1.10↑ 경보 |
| **빈도** | 사고건수 ÷ 노출계약수 | 심도와 분리하여 원인 판별 |
| **심도** | 총손해액 ÷ 사고건수 | 의료비 단가 상승 등 탐지 |
| **IBNR 조정 MLR** | (보고손해 + IBNR 추정액) ÷ 경과보험료 × 100 | 조기 과소 계상 보정 |

> 빈도·심도 분리 원칙: 손해율 상승 원인이 빈도 주도이면 가입 제한 강화, 심도 주도이면 보험료(요율) 조정 방향으로 UW 대응책을 차별화한다.

---

## 4. 분석 기법 선택 (담보 규모별)

| 담보 구분 | 적용 기법 | 근거 |
|---|---|---|
| **대형 담보** | EWMA(λ=0.2) + STL 계절분해 + SPC(I-MR 관리도) | 데이터 충분, 계절성 제거 후 순수 추세 진단 |
| **중형 담보** | EWMA(λ=0.3) + YoY 변화율 비교 | 구현 단순, 계절 효과 YoY로 통제 |
| **소형·신규 담보** | BF법 기반 추정 MLR + ELR ±15% 범위 | 데이터 희소 → 통계 경보 대신 계리 추정 활용 |
| **공통 (전 담보)** | Changepoint Detection (PELT) | 상품 개정·의료 환경 급변 구간 자동 탐지 |

---

## 5. 경보 등급 및 기준

### 5-1. 대형 담보

| 등급 | 조건 | 조치 |
|---|---|---|
| **주의** | MLR/ELR ≥ 1.05 또는 3개월 연속 상승 | 담당자 알림, 다음 정기 리뷰 시 검토 |
| **경보** | MLR/ELR ≥ 1.10 또는 MLR이 95 퍼센타일 초과 | 즉시 UW 담당자 검토 + 계리팀 공유 |
| **긴급** | MLR/ELR ≥ 1.20 또는 EWMA 3σ 이탈 | 에스컬레이션 (UW팀장·계리팀장 보고) |

### 5-2. 중형 담보

| 등급 | 조건 | 조치 |
|---|---|---|
| **주의** | MLR/ELR ≥ 1.08 | 담당자 확인 트리거 |
| **경보** | MLR/ELR ≥ 1.08 + 3개월 연속 | 정식 검토 요청 |

### 5-3. 소형·신규 담보

- 통계적 경보 기준 미적용
- BF법 추정 MLR이 ELR ±15% 초과 → 분기 리뷰 대상 지정
- 신규 담보: 12개월 경과 후 중형 기준 전환, 24개월 후 독립 통계 기준 전환

### 5-4. 이벤트 마스킹

- 자연재해(태풍·집중호우·폭설) 및 규정 변경 이벤트 발생 월: **이벤트 플래그** 부여
- 이벤트 플래그 월은 경보 판정에서 자동 제외, 기저 손해율(Underlying Loss Ratio) 별도 산출
- 이벤트 플래그 기준: 협회 공시 또는 감독기관 자료 기반으로 객관적 수립

## 6. 시스템 프로세스 설계 (안)

```
[입력]
  담보별 월별 (MLR, 빈도, 심도, 경과보험료, 청구건수)
  ELR 테이블 (요율 산출 기준)
  이벤트 마스킹 플래그 테이블

        ↓

[전처리]
  담보 규모 등급 분류
  이벤트 플래그 적용 (해당 월 경보 판정 제외)
  청구건수 30건 미만 담보 → 정성 리뷰 대상 분리

        ↓

[트렌드 분석]
  대형: EWMA + STL + SPC → MLR/ELR 비율 + 퍼센타일 판정
  중형: EWMA + YoY → MLR/ELR 비율 판정
  소형: BF법 추정 MLR → ELR ±15% 판정
  공통: PELT Changepoint Detection → 구조 변화 구간 식별

        ↓

[경보 판정]
  등급 산출 (정상 / 주의 / 경보 / 긴급)
  빈도·심도 원인 분류 (빈도 주도 / 심도 주도 / 복합)

        ↓

[알림 및 리뷰]
  자동 알림 (대시보드 + 이메일)
  UW 담당자 정성 검토 → 조치 여부 기록
  피드백 로그 축적 → 분기별 임계치 재검토
```

---

## 7. 입력 데이터 요건

| 데이터 | 집계 단위 | 최소 기간 | 비고 |
|---|---|---|---|
| 담보별 발생손해액 | 담보×월 | 24개월 | IBNR 조정 전·후 구분 |
| 경과보험료 | 담보×월 | 24개월 | |
| 청구건수 | 담보×월 | 24개월 | 빈도 산출용 |
| 노출계약수 | 담보×월 | 24개월 | 빈도 산출용 |
| ELR 테이블 | 담보 | 최신 차수 | 요율 산출 기준 동기화 필요 |
| 이벤트 마스킹 테이블 | 월 | 전 기간 | 협회·감독기관 기준 |

> 실제 계약·청구 원천 데이터 및 고객 식별 정보는 입력 금지 (CLAUDE.md 보안 규칙)
