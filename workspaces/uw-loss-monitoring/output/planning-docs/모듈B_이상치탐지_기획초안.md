## 1. 목적

모듈 A(트렌드 진단)가 완만한 추세 악화를 탐지하는 반면, 모듈 B는 **단기 급등·구조적 이탈 등 비정상 패턴을 자동으로 식별**하는 데 목적을 둔다.

- 단일 지표 감시가 아닌 빈도·심도·MLR 다변량 복합 탐지
- 비지도학습 기반으로 레이블 없이 운영 가능
- UW 담당자가 해석할 수 있는 원인 분류 결과 제공 (XAI 연계)

**모듈 A와의 역할 분담:**

| 구분 | 모듈 A (트렌드 진단) | 모듈 B (이상치 탐지) |
|---|---|---|
| 탐지 대상 | 완만한 추세 악화, 수준 전환 | 급격한 이탈, 비정상 패턴 |
| 탐지 시점 | 3개월 이상 누적 신호 | 단발·복합 이상 즉시 탐지 |
| 주요 기법 | EWMA, STL, SPC, BF법 | Isolation Forest, SARIMA 잔차, Prophet |
| 출력 형식 | 추세 리포트 + 경보 등급 | 이상 스코어 + 원인 변수 분류 |

---

## 2. 분석 대상 및 집계 기준

| 항목 | 기준 |
|---|---|
| **분석 단위** | 담보별 × 월별 |
| **기준일** | 매월 말일 (경과보험료 마감 기준) |
| **입력 변수** | 빈도, 심도, MLR, 청구건수, 경과보험료 (5개 다변량) |
| **최소 분석 기간** | 24개월 이상 |
| **제외 조건** | 청구건수 30건 미만 담보 → ML 탐지 제외, 추세 모니터링만 적용 |
| **전처리** | RobustScaler 표준화 (이상치에 강건), 이벤트 마스킹 플래그 적용 |

---

## 3. 알고리즘 선택

### 3-1. 1차 탐지: 시계열 기반 (설명 가능)

| 알고리즘 | 적용 방법 | 경보 조건 |
|---|---|---|
| **SARIMA 잔차 분석** | 담보별 MLR 시계열 모델링 → 잔차 정규성 검정 | 잔차 3σ 초과 시 1차 플래그 |
| **Prophet 예측구간 이탈** | 월별 MLR 예측 → 실제값과 예측 신뢰구간 비교 | 80% 구간 이탈 → 주의 / 95% 구간 이탈 → 플래그 |

> 1차 탐지는 해석이 직관적이며 UW 담당자에게 바로 설명 가능한 결과를 제공한다.

### 3-2. 2차 탐지: 다변량 ML (정밀 분석)

| 알고리즘 | 적용 방법 | 경보 조건 |
|---|---|---|
| **Isolation Forest** | 담보별 (빈도, 심도, MLR, 청구건수, 경과보험료) 다변량 입력 → 이상 스코어 산출 | 이상 스코어 임계치 초과 |
| **LOF (Local Outlier Factor)** | 유사 담보군 내 국소 밀도 비교 → 상대적 이상도 산출 | LOF 스코어 임계치 초과 |

> 2차 탐지는 단일 변수로 포착하기 어려운 복합 이상 패턴을 탐지한다.

### 3-3. 앙상블 의결 기준

```
1차 탐지 (SARIMA 또는 Prophet) + 2차 탐지 (Isolation Forest 또는 LOF)
→ 동시 탐지 시: 정식 알림 발송
→ 1개만 탐지 시: 주의 등급 (모니터링 강화)
→ 3개월 연속 주의 등급: 경보 등급으로 자동 상향
```

---

## 4. XAI 연계: 원인 분류

이상 탐지 결과를 UW 담당자가 해석할 수 있도록 원인 변수를 분류하여 제공한다.

### 4-1. SHAP 기반 원인 기여도 산출

- Isolation Forest 결과에 SHAP TreeExplainer 적용
- 이상 스코어를 높인 변수별 기여도를 수치로 산출
- 예시: "청구건수 급증(기여 60%) + 심도 증가(기여 30%)"

### 4-2. 이상 원인 3분류

| 유형 | 정의 | UW 대응 방향 |
|---|---|---|
| **빈도 이상** | 청구건수·사고 발생률 급등, 심도는 정상 | 가입 자격 기준 강화, 모집 채널 점검 |
| **심도 이상** | 건당 지급액 급등, 청구건수는 정상 | 보험료(요율) 조정, 고액 청구 심사 강화 |
| **복합 이상** | 빈도·심도 동시 이탈 | 담보 구조 재검토, 에스컬레이션 |

### 4-3. Alert Card 출력 형식

```
[이상 탐지 알림]
기간: 2026년 4월
담보: [입원의료비]
이상 유형: 빈도 이상
탐지 모델: SARIMA 잔차 3σ 이탈 + Isolation Forest 동시 탐지
원인 변수: 청구건수 전월 대비 +42% (기여도 65%), 건당 보험금 정상 범위 내
권고 조치: 청구 패턴 및 모집 채널 점검 요청
```

## 5. False Positive 관리

| 기법 | 내용 |
|---|---|
| **앙상블 의결** | 1차+2차 동시 탐지 시에만 정식 알림 → 단일 모델 오탐 필터링 |
| **연속 N회 조건** | 단발 탐지는 주의 등급, 3개월 연속 탐지 시 경보 등급 상향 |
| **이벤트 마스킹** | 자연재해·규정 변경 발생 월 자동 억제 (모듈 A와 동일 플래그 공유) |
| **담보별 개별 임계치** | 소형 담보 완화, 대형 담보 엄격 — 담보 등급별 차등 임계치 테이블 |
| **Human-in-the-loop** | UW 담당자 피드백(정상 확인·이상 확인·보류) → 임계치 자동 재보정 |

## 6. 시스템 프로세스 설계 (안)

```
[입력]
  담보별 월별 (빈도, 심도, MLR, 청구건수, 경과보험료)
  이벤트 마스킹 플래그 (모듈 A 공유)

        ↓

[전처리]
  RobustScaler 표준화
  청구건수 30건 미만 담보 분리 (ML 탐지 제외)
  이벤트 플래그 월 억제 처리

        ↓

[1차 탐지: 시계열 기반]
  SARIMA 잔차 3σ 이탈 판정
  Prophet 95% 예측구간 이탈 판정
  → 1개 이상 탐지 시 1차 플래그

        ↓

[2차 탐지: 다변량 ML]
  Isolation Forest 이상 스코어 산출
  LOF 이상 스코어 산출
  → 1개 이상 임계치 초과 시 2차 플래그

        ↓

[앙상블 의결]
  1차 + 2차 동시 플래그 → 정식 알림 (경보)
  1차 또는 2차만 → 주의 등급
  3개월 연속 주의 → 경보 자동 상향

        ↓

[XAI 레이어]
  SHAP 기반 원인 변수 기여도 산출
  빈도 이상 / 심도 이상 / 복합 이상 분류
  Alert Card 생성

        ↓

[Human-in-the-loop]
  UW 담당자 피드백 (정상 확인 / 이상 확인 / 보류)
  피드백 로그 적재 → 분기별 임계치 재보정
```

## 7. 모듈 A 연계 설계

모듈 A(트렌드 진단)와 모듈 B(이상치 탐지)는 다음 요소를 공유한다.

| 공유 항목 | 내용 |
|---|---|
| **이벤트 마스킹 플래그** | 동일 테이블 참조 — 일관된 예외 처리 |
| **담보 규모 등급** | 동일 분류 기준 적용 — 임계치 차등화 |
| **경보 로그** | 통합 대시보드에서 A·B 경보 이력 동시 조회 |
| **피드백 루프** | UW 담당자 피드백을 A·B 임계치 재보정에 공동 활용 |

**통합 판정 원칙:**
- 모듈 A 경보 + 모듈 B 경보 동시 발생 → **긴급** 등급 에스컬레이션
- 모듈 A 주의 + 모듈 B 경보 → 경보 유지, 빈도·심도 원인 분류 우선 확인
