## 4. 제약 조건 및 특이사항

### 4.1 개인정보 처리

- 실데이터는 반드시 마스킹 완료 후 입력한다.
- 고객명, 주민등록번호, 증권번호 원문, 청약번호 원문, 연락처, 주소, 계약자/피보험자 식별 가능 정보는 포함하지 않는다.
- M-04 사고 내용(accident_description) 컬럼은 개인식별 가능 표현 제거 후 사용한다.
- 설계사 코드(agent_code)는 비식별 내부 코드로만 처리한다.

### 4.2 연동 및 저장 위치

- 저장 위치: `workspaces/uw-report-automation/`
- 연동 시스템: 미정 (실데이터 제공 시 확정)

### 4.3 기타 업무 상 제약

- 현재 실데이터 미제공 상태이므로 모델 설계안 완성 후 실데이터 연동 단계를 별도 진행한다.
- M-04 위험도 스코어링 모델의 가중치 및 임계값은 실데이터 기반 검증 후 최종 확정한다.

## 5. 후속 Agent 전달 사항

### 5.1 uw-model-builder 전달 핵심 인수 기준

| 모델 | 전달 기준 |
|---|---|
| M-01 손해율 | 집계 단위: 상품코드 × 계약연월 / 이상 감지 기준: 손해율 임계값 (실데이터 제공 시 확정) / 기간: 월별 추이 최소 12개월 권장 |
| M-02 유지율 | 집계 단위: 계약 코호트(가입월 기준) / 분석 기준: 경과 월수별 해지율 / 생존 분석 적용 여부 검토 |
| M-03 심사데이터 | 집계 단위: 심사일 × 상품코드 / 결과 분류: 승인/조건부/거절/보류 / 소요일 효율 지표 포함 |
| M-04 취급자 위험 | 집계 단위: 설계사 코드 / 스코어링 변수: 손해율·청구집중도·조기해지율·병원집중도·고액계약비율 등 / 등급 분류 기준: 설계 후 확정 |
