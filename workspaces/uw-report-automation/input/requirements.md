## 1. 분석 목적

### 1.1 분석 목적 분류

| 모델 번호 | 모델명 | 분석 목적 |
|---|---|---|
| M-01 | 손해율 분석 | 기간별/상품별 손해율 추이 집계 및 이상 감지 |
| M-02 | 유지율 분석 | 기간별/상품별 계약 유지/해지 패턴 분석, 사고데이터 및 손해율 데이터와 연계하여 유형/패턴 감지 |
| M-03 | 심사데이터 분석 | UW 심사 결과 분포 및 심사 효율 분석 |
| M-04 | 보험취급자 위험 관리 | 설계사(취급자) 다차원 위험도 스코어링 및 등급 분류, 손해 증가 유형/패턴 감지 |

### 1.2 보고 대상

- 내부 UW 팀
- 경영진 보고용

### 1.3 산출물 범위

- 모델 설계안 (대시보드 불필요)
- 실데이터 제공 예정 (마스킹 처리 후 사용) — 현재 미제공 상태

## 4. 제약 조건 및 특이사항

### 4.1 개인정보 처리

- 실데이터는 반드시 마스킹 완료 후 입력한다.
- 고객명, 주민등록번호, 증권번호 원문, 청약번호 원문, 연락처, 주소, 계약자/피보험자 식별 가능 정보는 포함하지 않는다.
- M-04 사고 내용(accident_description) 컬럼은 개인식별 가능 표현 제거 후 사용한다.
- 설계사 코드(agent_code)는 비식별 내부 코드로만 처리한다.

### 4.2 연동 및 저장 위치

- 저장 위치: `workspaces/uw-report-automation/`
- 연동 시스템: 미정 (실데이터 제공 시 확정)

### 4.3 기타 업무 상 제약

- 현재 실데이터 미제공 상태이므로 모델 설계안 완성 후 실데이터 연동 단계를 별도 진행한다.
- M-04 위험도 스코어링 모델의 가중치 및 임계값은 실데이터 기반 검증 후 최종 확정한다.
