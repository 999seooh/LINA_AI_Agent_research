## M-01 손해율 분석 모델

### 1. 목적

#### 1.1 분석 목적

청약월 기준 코호트 × 경과월 발전삼각형 구조로 손해율 추이를 집계하고, 위험보험료 기준(LR(riskP))과 총보험료 기준(LR(GWP)) 이중 손해율을 병행 산출한다. 담보 3계층·조직 3계층·유의취급자/피보험자 분리 집계를 통해 UW 팀의 포트폴리오 리스크 관리 및 경영 보고 기반을 제공한다.

#### 1.2 모델 적용 범위

| 항목 | 내용 |
|---|---|
| 분석 단위 | inceptionYM(청약월) × Elapsedmonth(경과월) × closeym(마감월) |
| 집계 기간 | closeym 기준 스냅샷 (발전삼각형 누적 구조) |
| 주요 분석 관점 | 이중 손해율(LR(riskP)/LR(GWP)), 발전삼각형 IBNR 추이, 담보·조직·채널·유의계약 분리 집계 |
| 기대 효과 | IBNR 포함 최종 손해율 조기 감지, 담보군별·조직별 리스크 차별화 관리, 경영 보고 기준 표준화 |

### 3. 변경/설계안

#### 3.1 모델 유형

통계 기반 집계 + 발전삼각형 + 이중 손해율 + 이상 감지 (Statistical Aggregation + Loss Development Triangle + Dual Loss Ratio + Anomaly Detection)

#### 3.2 집계 기준

| 항목 | 기준 |
|---|---|
| 기준 시점 | closeym(마감월, YYYYMM) 기준 월말 스냅샷 |
| 코호트 기준 | inceptionYM(청약월, YYYYMM) — 신계약 모집 시점 |
| 경과 기간 축 | Elapsedmonth — 발전삼각형의 열(Column) 축 |
| 기본 집계 단위 | inceptionYM × Elapsedmonth × closeym |
| 상위 집계 차원 | Uw_productgroup_G(상품대분류) / Uw_treatygroup_seq(담보대분류) / newchannel(채널) |
| 조직 드릴다운 | groupId_groupKorName(대리점) → dept_groupKorName(지사) → team_groupKorName(지점) |
| 유의계약 분리 | checksellerId is not Null / checkInsuredIdkey is not Null 기준 서브셋 분리 집계 |
| 필터 조건 | riskP > 0인 유효 계약 기준 (실데이터 제공 시 확정) |

#### 3.3 핵심 지표 정의

**이중 손해율 지표 (핵심)**

| 지표명 | 산식 | 설명 |
|---|---|---|
| LR(riskP) | sum(loss_Amt) / sum(riskP) × 100 | 위험보험료 기준 손해율. 순수 위험 대비 지급 비율. |
| LR(GWP) | sum(ult_loss_Amt) / (sum(GWP) - sum(ROP)) × 100 | 총보험료(만기환급금 제외) 기준. IBNR 보정 포함. 경영 보고 기준. |
| LR(riskP) 전월 대비 변화율 | (당월 LR(riskP) - 전월 LR(riskP)) / 전월 LR(riskP) × 100 | 추이 이상 감지 보조 |
| LR(GWP) 전월 대비 변화율 | (당월 LR(GWP) - 전월 LR(GWP)) / 전월 LR(GWP) × 100 | 경영 기준 추이 이상 감지 보조 |
| 3개월 이동평균 LR(riskP) | 최근 3개월 LR(riskP) 단순 평균 | 노이즈 제거 후 추세 확인 |
| 3개월 이동평균 LR(GWP) | 최근 3개월 LR(GWP) 단순 평균 | 노이즈 제거 후 추세 확인 |

**빈도·심도 보조 지표**

| 지표명 | 산식 | 설명 |
|---|---|---|
| 청구 빈도 (Frequency) | sum(claims) / sum(exposure) | 증권 1건당 청구금액. 빈도 집중 감지. |
| 청구 심도 (Severity) | sum(loss_Amt) / sum(claims) | 청구 1건당 평균 보험금. 고액 청구 감지. |
| IBNR 조정 비율 | sum(ult_loss_Amt) / sum(loss_Amt) - 1 | 기발생미보고 보정 배수. 값이 클수록 미보고 부담 증가. |
| IBNR 조정 청구 비율 | sum(ult_claims) / sum(claims) - 1 | 청구 기준 IBNR 보정 배수. |

**유의계약 분리 손해율**

| 지표명 | 집계 조건 | 설명 |
|---|---|---|
| 유의취급자 LR(riskP) | checksellerId is not Null 기준 서브셋 | 유의취급자 계약의 위험보험료 기준 손해율 |
| 유의피보험자 LR(riskP) | checkInsuredIdkey is not Null 기준 서브셋 | 유의피보험자 계약의 위험보험료 기준 손해율 |
| 일반계약 LR(riskP) | 두 플래그 모두 Null인 서브셋 | 일반 계약 손해율 (비교 기준선) |
| 유의계약 LR 배수 | 유의취급자 LR(riskP) / 일반계약 LR(riskP) | 일반 대비 유의계약 손해율 배율. 역선택 수준 확인. |

#### 3.4 발전삼각형 구조 (Loss Development Triangle)

**매트릭스 구조**

- 행(Row): inceptionYM (청약연월, 계약 코호트)
- 열(Column): Elapsedmonth (경과월 번호)
- 마감 시점: closeym (특정 마감월 기준 스냅샷)
- 셀(Cell): 해당 코호트 × 경과월 기준 LR(GWP) (%)

**발전삼각형 예시 구조**

| 청약월(inceptionYM) | 경과 3M | 경과 6M | 경과 12M | 경과 24M |
|---|---|---|---|---|
| 202301 | XX.X% | XX.X% | XX.X% | XX.X% |
| 202302 | XX.X% | XX.X% | XX.X% | - |
| 202303 | XX.X% | XX.X% | - | - |
| ... | ... | ... | ... | ... |

> closeym 기준 스냅샷으로 삼각형을 누적 관리한다. 경과월이 진행될수록 ult_loss_Amt 안정화 여부를 추적한다.

**분석 차원 확장**

| 차원 | 분리 기준 | 목적 |
|---|---|---|
| 전체 | 전체 코호트 | 기준선 손해율 |
| 채널별 | newchannel (GA/CM/OB) | 채널별 손해율 성과 비교 |
| 담보군별 | Uw_treaty_G | 담보별 손해율 분리 (핵심 분석 축) |
| 담보대분류별 | Uw_treatygroup_seq | 담보 최상위 분류 롤업 |
| 상품구분별 | Uw_product_G | 상품 단위 분리 |
| 연령대별 | agegroup2 | 연령 구간별 손해율 패턴 |
| 조직별 | 대리점 → 지사 → 지점 | 조직 단위 드릴다운 |

#### 3.5 이상 감지 로직

**임계값 기반 이상 감지 (Rule-based Alert)**

| 감지 조건 | 기준값 | 처리 방법 |
|---|---|---|
| LR(riskP) 단일 월 초과 | TBD% (실데이터 분포 확인 후 확정) | 해당 담보군·코호트 플래그 처리 |
| LR(GWP) 단일 월 초과 | TBD% (실데이터 분포 확인 후 확정) | 경영 보고 기준 이상 플래그 |
| LR(riskP) 전월 대비 급등 | +15%p 초과 (검토 필요) | 변화율 이상 플래그 |
| LR(GWP) 3개월 연속 상승 | 3개월 연속 증가 | 추세 이상 플래그 |
| 청구 빈도 급증 | 전월 대비 +30% 초과 (검토 필요) | 빈도 이상 플래그 |
| IBNR 조정 비율 이상 | IBNR 배수 TBD 초과 (검토 필요) | 미보고 집중 플래그 |
| 유의취급자 LR 배수 임계 초과 | 일반 대비 TBD배 초과 (검토 필요) | 유의계약 집중 플래그 |

> 임계값은 모두 "검토 필요" 상태로, 실데이터 분포 확인 후 최종 확정한다.

**통계 기반 이상 감지 (Statistical Outlier)**

| 방법 | 기준 | 설명 |
|---|---|---|
| Z-score | \|Z\| > 2.5 (검토 필요) | 담보군 × 코호트 단위 표준화 후 이상치 판별 |
| IQR 방법 | Q3 + 1.5×IQR 초과 | LR(riskP) 분포 상단 이상치 탐지 |

#### 3.6 집계 흐름

```
[원천 데이터]
  → 유효 계약 필터 (riskP > 0)
  → 유의취급자 / 유의피보험자 / 일반 서브셋 분리
  → inceptionYM × Elapsedmonth × closeym 단위 집계
  → LR(riskP) 산출: sum(loss_Amt) / sum(riskP) × 100
  → LR(GWP) 산출: sum(ult_loss_Amt) / (sum(GWP) - sum(ROP)) × 100
  → 빈도(Frequency) / 심도(Severity) 산출
  → IBNR 조정 비율 산출
  → 이동평균·변화율 파생 지표 산출
  → 임계값 비교 → 이상 플래그 생성
  → 담보군(Uw_treaty_G) / 채널(newchannel) / 조직 드릴다운 집계
  → 유의계약 LR 배수 산출
  → 발전삼각형 매트릭스 출력
  → 최종 출력 테이블
```

### 4. 테스트 기준

| 테스트 항목 | 기준값 | 검증 방법 | 담당 |
|---|---|---|---|
| LR(riskP) 산출 정확성 | sum(loss_Amt) / sum(riskP) × 100 수식 일치 | 샘플 담보군·코호트 5건 수동 계산 대비 | 검토 필요 |
| LR(GWP) 산출 정확성 | sum(ult_loss_Amt) / (sum(GWP) - sum(ROP)) × 100 수식 일치 | 동일 샘플 수동 계산 대비 | 검토 필요 |
| 발전삼각형 완전성 | inceptionYM × Elapsedmonth 조합 누락 0건 | 원천 코호트 목록 대비 확인 | 검토 필요 |
| closeym 스냅샷 정합성 | 동일 closeym 기준 집계 건수 = 원천 건수 | 마감월별 집계 합산 대조 | 검토 필요 |
| 이중 손해율 방향 일관성 | LR(GWP) >= LR(riskP) 여부 (IBNR 포함 시 일반적으로 성립) | 전체 코호트 비교 분포 확인 | 검토 필요 |
| IBNR 조정 비율 범위 | sum(ult_loss_Amt) >= sum(loss_Amt) (역전 건 확인) | 역전 케이스 별도 검토 | 검토 필요 |
| 빈도·심도 산출 정확성 | Frequency = sum(claims)/sum(exposure), Severity = sum(loss_Amt)/sum(claims) | 샘플 5건 수동 계산 대비 | 검토 필요 |
| 유의계약 분리 완전성 | 일반 + 유의취급자 + 유의피보험자 건수 합산 >= 전체 건수 (중복 포함 가능) | 분리 집계 건수 대비 | 검토 필요 |
| 이상 감지 플래그 정확성 | 임계값 초과 건 플래그 누락률 0% | 조건 수동 검증 | 검토 필요 |
| 담보군 분리 완전성 | Uw_treaty_G 전체 목록 커버리지 100% | 원천 담보군 목록 대비 확인 | 검토 필요 |
| 조직 드릴다운 정합성 | 지점 합산 = 지사 합산 = 대리점 합산 | 계층 집계 합계 검증 | 검토 필요 |
| 채널 분리 완전성 | GA/CM/OB 전체 커버리지 100% | 원천 채널 목록 대비 확인 | 검토 필요 |

### 5. 기대효과

| 항목 | 기대 내용 |
|---|---|
| IBNR 포함 경영 기준 손해율 제공 | LR(GWP) 산출로 미보고 손해를 반영한 최종 손해율 경영 보고 기준 표준화 |
| 위험보험료 기준 순수 리스크 파악 | LR(riskP) 산출로 할증·부담보 등 인수 조건 반영 순수 위험 대비 지급 수준 모니터링 |
| 담보군별 리스크 차별화 관리 | Uw_treaty_G 3계층 분리 집계로 손해 집중 담보 식별 및 인수 기준 재검토 근거 제공 |
| 발전삼각형 IBNR 추이 관리 | inceptionYM × Elapsedmonth 매트릭스로 코호트별 손해 발전 패턴 및 IBNR 안정화 시점 파악 |

## M-02 유지율 분석 모델

### 1. 목적

#### 1.1 분석 목적

청약월 기준 코호트별 MMP(월납보험료) 금액 기준 유지율을 집계하고, 주요 회차(4·7·13·25회차)별 이탈 패턴을 추적하여 채널·상품별 유지 성과를 비교한다. 유지율이 낮은 코호트·채널의 손해 지표(loss_Amt, riskP)를 동시 추적하여 유지 부진과 손해 간의 연계 리스크를 관리한다.

#### 1.2 모델 적용 범위

| 항목 | 내용 |
|---|---|
| 분석 단위 | 청약월(코호트) × 기준년월 매트릭스 |
| 유지율 산식 | 유지MMP / 모집MMP × 100 (금액 기준) |
| 주요 분석 관점 | 코호트별 MMP 유지율, 주요 회차 이탈 패턴, 채널·상품별 유지 성과, 손해 연계 분석 |
| 기대 효과 | 조기 이탈 회차 식별, 유지율 취약 채널·상품군 관리, 유지율-손해율 연계 감지 |

### 3. 변경/설계안

#### 3.1 모델 유형

코호트 분석 + MMP 기반 유지율 집계 (Cohort Analysis + MMP-based Retention Aggregation)

- 계약 단건 lapse_yn 기반 생존 분석(Kaplan-Meier) 구조를 폐기
- 집계된 MMP 금액(모집MMP / 유지MMP) 기준 코호트 유지율 산출 구조로 전면 재설계
- 주요 회차(회차구분) 기준 이탈 패턴 집중 분석 병행

#### 3.2 집계 기준

| 항목 | 기준 |
|---|---|
| 기준일 | 기준년월(YYYYMM) 기준 월말 마감 |
| 코호트 기준 | 청약월(YYYYMM) — 신계약 모집 시점 |
| 경과 기간 산출 | (기준년월 - 청약월) 월 차이 계산 |
| 집계 단위 | 청약월 × 기준년월 × 채널 × 상품코드 |
| 상위 집계 | 상품군 / 상품그룹 단위 롤업 집계 병행 |
| 조직 드릴다운 | agency명 → center명 단계적 집계 |
| 필터 조건 | 모집MMP > 0인 유효 코호트 기준 (이상 데이터 제외 조건은 실데이터 확인 후 확정) |

#### 3.3 핵심 지표 정의

**MMP 기반 유지율 지표**

| 지표명 | 산식 | 설명 |

| 코호트 누적 유지율 | 기준년월 기준 청약월별 누적 유지율 | 코호트 경과 기간별 잔존율 |
| 이탈 MMP | 모집MMP - 유지MMP | 절대 금액 기준 이탈 규모 |

**주요 회차별 유지율 지표**

| 25회차 | 2년 납입 완료 시점 | 장기 유지율 기준 — 2년 코호트 잔존 성과 |

**채널·상품별 유지 성과 비교 지표**

| 지표명 | 집계 기준 | 설명 |
|---|---|---|
| 채널별 MMP 유지율 | 채널 × 청약월 × 기준년월 | GA/CM/OB 채널 간 유지 성과 비교 |

### 1. 목적

#### 1.1 분석 목적

UW 심사통계 SQL 기반의 실제 확정 컬럼을 활용하여, 자동/인수심사 건수 추이, 알릴사항 고지 패턴, KCD 상병 집중도, 거절 사유 텍스트 분석, 채널·상품별 심사 분포를 집계·분석한다. 이를 통해 심사 품질 모니터링 및 인수 기준 고도화 근거를 제공한다.

#### 1.2 모델 적용 범위

| 항목 | 내용 |
|---|---|
| 분석 단위 | 자동심사월(YYYYMM) × 상품코드 × 채널 |
| 심사 유형 분류 | 자동심사 / 인수심사 (자동/인심사 컬럼 기반) |
| 주요 분석 관점 | 심사 유형 비율 추이, 알릴사항 고지 패턴, KCD 상병 집중도, 거절 패턴, 채널·상품별 분포, 피보험자 프로파일 |
| 기대 효과 | 심사 유형 효율 모니터링, 고지 항목 기반 인수 기준 고도화 근거 제공 |

#### 1.3 활용 컬럼

| 컬럼명 | 업무적 의미 | 활용 방식 |
|---|---|---|
| 증권번호 | 계약 식별 키 | 건수 집계 전용. 출력물 포함 금지 |
| 청약번호 | 청약 식별 키 | 건수 집계 전용. 출력물 포함 금지 |
| 성별 | 피보험자 성별 (남/여) | 피보험자 프로파일 분석 |
| 연령 | 연령 구간 (00-09세 ~ 71세 이상) | 연령대별 심사 패턴 분석 |
| 자동심사월 | 심사 집계 기준 월(YYYYMM) | 집계 기준일 |
| 자동심사일 | 심사 처리 일자(YYYYMMDD) | 일별 집계 보조 |
| 상품코드 | 상품 코드 | 상품별 분리 집계 기준 |
| 상품구분 | Uw_product_G | 상품 상위 분류 집계 |
| 상품군 | Uw_productGroup_G | 상품군 롤업 집계 |
| 플랜코드 | 플랜 코드 | 플랜 단위 분리 집계 |
| 플랜군 | 플랜 구분 | 플랜 상위 분류 집계 |
| 채널 | GA / CM / OB / 기타 (파생) | 채널별 분리 집계 기준 |
| 자동/인심사 | 자동심사 또는 인수심사 구분 | 심사 유형 분류 핵심 기준 |
| 알릴사항1 | 고지 항목 1 | 고지 패턴 빈도 분석 |
| 알릴사항1세부내역 | 고지 항목 1 세부 텍스트 | 세부 고지 텍스트 분석 보조 |
| 알릴사항2 | 고지 항목 2 | 고지 패턴 빈도 분석 |
| 알릴사항2세부내역 | 고지 항목 2 세부 텍스트 | 세부 고지 텍스트 분석 보조 |
| 알릴사항3 | 고지 항목 3 | 고지 패턴 빈도 분석 |
| 알릴사항3세부내역 | 고지 항목 3 세부 텍스트 | 세부 고지 텍스트 분석 보조 |
| 알릴사항KCD코드 | 고지 항목 KCD 상병 코드 | 상병 코드 집중도 분석 |
| 알릴사항KCD명 | 고지 항목 KCD 상병명 | 상병명 기반 분류 보조 |
| UW메모 | UW 심사 메모 (자유 텍스트) | 개인식별 표현 제거 후 키워드 분석 |
| 거절메모 | 거절 사유 메모 (자유 텍스트) | 개인식별 표현 제거 후 거절 패턴 분석 |
| 요청메모 | 심사 요청 메모 (자유 텍스트) | 개인식별 표현 제거 후 요청 패턴 분석 보조 |

> 증권번호·청약번호는 건수 카운트 집계에만 사용하며, 분석 출력물에 원문을 포함하지 않는다.
> UW메모·거절메모·요청메모는 개인식별 가능 표현(성명, 주민번호 등) 제거 후 키워드 분석에 한해 활용한다.

### 3. 변경/설계안

#### 3.1 모델 유형

분포 집계 + 텍스트 빈도 분석 (Distribution Aggregation + Text Frequency Analysis)

- 자동/인심사 구분 기반 심사 유형 비율 통계 집계
- 알릴사항 항목별 빈도 및 KCD 코드 집중도 집계
- 거절메모·UW메모 개인식별 제거 후 키워드 빈도 분석

#### 3.2 집계 기준

| 항목 | 기준 |
|---|---|
| 기준일 | 자동심사월(YYYYMM) 기준 월말 마감 |
| 집계 단위 | 자동심사월 × 상품코드 × 채널 |
| 상위 집계 | 상품구분(Uw_product_G) 및 상품군(Uw_productGroup_G) 단위 롤업 집계 병행 |
| 채널 분리 | 채널 컬럼 기준 GA / CM / OB / 기타 서브 집계 |
| 필터 조건 | 유효 심사 건 기준 (취소·무효 제외 조건은 실데이터 제공 시 확정) |
| 식별 정보 처리 | 증권번호·청약번호는 COUNT 집계에만 사용. 텍스트 컬럼은 개인식별 표현 제거 후 활용 |

#### 3.3 핵심 지표 정의

**자동/인심사 비율 지표**

| 지표명 | 산식 | 설명 |
|---|---|---|
| 자동심사 비율 | 자동심사 건수 / 전체 심사 건수 × 100 | 자동 처리 효율 추이 모니터링 |
| 인수심사 비율 | 인수심사 건수 / 전체 심사 건수 × 100 | 수동 심사 집중 구간 식별 |
| 채널별 자동심사 비율 | 채널 × 자동/인심사 조합 건수 / 채널 전체 건수 × 100 | 채널별 처리 유형 패턴 비교 |
| 상품별 인수심사 집중도 | 상품코드별 인수심사 건수 / 상품코드별 전체 건수 × 100 | 수동 심사 집중 상품 식별 |
| 월별 유형 전환 추이 | 자동심사월 기준 자동/인심사 비율 월별 변화 | 심사 유형 비율 변화 감지 |

**알릴사항 고지 패턴 지표**

| 지표명 | 산식 | 설명 |
|---|---|---|
| 알릴사항1 항목별 빈도 | 알릴사항1 값별 건수 / 전체 건수 × 100 | 가장 빈번한 고지 항목 Top-N 랭킹 |
| 알릴사항2 항목별 빈도 | 알릴사항2 값별 건수 / 알릴사항2 입력 건수 × 100 | 2차 고지 패턴 분포 |
| 알릴사항3 항목별 빈도 | 알릴사항3 값별 건수 / 알릴사항3 입력 건수 × 100 | 3차 고지 패턴 분포 |
| 고지 항목 복수 입력 비율 | 알릴사항2 이상 입력 건수 / 전체 건수 × 100 | 복수 고지 집중 구간 확인 |
| 상위 고지 항목 랭킹 | 알릴사항1~3 통합 항목별 빈도 합산 Top-20 | 전체 고지 패턴 요약 |

**KCD 상병 집중도 지표**

| 지표명 | 산식 | 설명 |
|---|---|---|
| KCD 코드별 건수 | 알릴사항KCD코드 값별 건수 집계 | 상병 코드 빈도 순위 |
| 상위 KCD 코드 집중도 | 상위 10개 KCD 코드 건수 / 전체 KCD 입력 건수 × 100 | 특정 상병 집중 여부 확인 |
| 상병군별 집계 | KCD 코드 앞 3자리 기준 대분류 그룹화 후 빈도 집계 | 상병군 단위 집중도 분석 |
| 채널 × KCD 조합 분포 | 채널별 상위 KCD 코드 빈도 분리 집계 | 채널별 고지 상병 패턴 비교 |

**거절 패턴 지표 (텍스트 기반)**

| 지표명 | 산식/방법 | 설명 |
|---|---|---|
| 거절메모 키워드 빈도 | 개인식별 제거 후 키워드 단위 토큰화 → 상위 빈도 키워드 Top-20 | 거절 사유 주요 패턴 파악 |
| 거절메모 키워드 Top-N | 월별 거절메모 내 상위 빈도 키워드 추이 | 거절 사유 패턴 변화 감지 |
| UW메모 키워드 빈도 | 동일 처리 방법 적용 | UW 판단 근거 주요 키워드 파악 |

> 텍스트 분석 전 반드시 성명, 주민등록번호, 연락처 등 개인식별 가능 패턴을 마스킹·제거한 후 분석에 활용한다.

**채널·상품별 분리 지표**

| 지표명 | 집계 기준 | 설명 |
|---|---|---|
| 채널 × 상품구분 × 자동/인심사 건수 분포 | 채널 × 상품구분 × 자동/인심사 3-way 조합 | 채널·상품별 심사 유형 패턴 비교 |
| 채널 × 자동심사월 심사 건수 추이 | 채널 × 자동심사월 집계 | 채널별 심사 볼륨 월별 추이 |
| 상품군별 알릴사항 고지 빈도 | 상품군 × 알릴사항1 항목 집계 | 상품 특성별 고지 패턴 차이 확인 |

**연령·성별 분포 지표**

| 지표명 | 집계 기준 | 설명 |
|---|---|---|
| 연령 구간별 심사 건수 비율 | 연령 구간(00-09세 ~ 71세 이상) × 전체 건수 | 피보험자 연령 분포 확인 |
| 성별 심사 건수 비율 | 성별(남/여) × 전체 건수 | 피보험자 성별 분포 확인 |
| 연령 × 성별 × 자동/인심사 분포 | 3-way 조합 집계 | 프로파일별 심사 유형 패턴 |
| 연령 × 알릴사항1 주요 고지 패턴 | 연령 구간 × 알릴사항1 항목 집계 | 연령대별 주요 고지 항목 비교 |

#### 3.4 후행 손해율 연계 (M-01 조인 방안)

> 현재 M-03 심사통계 데이터와 M-01 손해율 데이터 간 직접 조인 키(증권번호 등)는 출력물에서 사용하지 않는다.
> 조인은 집계 단위(상품코드 × 자동심사월 × 채널) 수준의 매핑으로 설계하며, 원건 식별 키를 분석 결과에 노출하지 않는다.

| 연계 항목 | 연계 기준 | 목적 |
|---|---|---|
| 자동심사 건 비율 × 후행 손해율 | 상품코드 × 자동심사월 집계 단위 조인 | 자동심사 처리 비율과 손해율 상관 확인 |
| 상위 고지 항목 × 후행 손해율 | 알릴사항1 주요 항목 × 상품코드 단위 조인 | 고지 항목별 실제 손해율 수준 파악 |
| 상위 KCD 코드 × 후행 손해율 | KCD 대분류 × 상품코드 단위 조인 | 상병군별 인수 리스크 수준 검토 |
| 채널별 심사 패턴 × 손해율 | 채널 × 자동심사월 단위 조인 | 채널별 심사 품질과 손해율 연관성 분석 |

### 4. 테스트 기준

| 테스트 항목 | 기준값 | 검증 방법 | 담당 |
|---|---|---|---|
| 자동/인심사 분류 완전성 | 미분류 건 0% | 자동/인심사 컬럼 전수 확인 | 검토 필요 |
| 자동/인심사 비율 합산 | 자동 + 인수심사 = 100% | 비율 합산 검증 | 검토 필요 |
| 자동심사월 기준 집계 정합성 | 월별 집계 건수 합산 = 전체 원천 건수 | 원천 대비 집계 완전성 확인 | 검토 필요 |
| 알릴사항 항목 빈도 집계 정확성 | 항목별 건수 합산 = 알릴사항 입력 전체 건수 | 합산 일치 검증 | 검토 필요 |
| KCD 코드 집계 완전성 | KCD 입력 건 중 미집계 건 0% | 원천 KCD 코드 목록 대비 확인 | 검토 필요 |
| 텍스트 개인식별 제거 | 거절메모·UW메모 내 성명·주민번호 패턴 잔존 0건 | 마스킹 처리 후 패턴 매칭 전수 확인 | 보안 담당 확인 필요 |
| 채널 분리 완전성 | GA/CM/OB/기타 전체 커버리지 100% | 원천 채널 목록 대비 확인 | 검토 필요 |
| 상품코드 분리 완전성 | product_code 전체 커버리지 100% | 원천 상품 목록 대비 확인 | 검토 필요 |
| M-01 조인 정합성 | 조인 기준(상품코드 × 자동심사월 × 채널) 불일치 건 확인 | 조인 결과 집계 단위 대조 | 실데이터 제공 시 확정 |
| 증권번호·청약번호 미노출 | 분석 출력물 내 원문 포함 0건 | 출력 컬럼 목록 전수 확인 | 보안 담당 확인 필요 |

### 5. 기대효과

| 항목 | 기대 내용 |
|---|---|
| 심사 유형 효율 모니터링 | 자동/인심사 비율 추이 자동 집계로 수동 심사 집중 구간 조기 식별 |
| 고지 패턴 기반 인수 기준 고도화 | 알릴사항 항목별 빈도 분석으로 주요 고지 항목 기반 인수 룰 재검토 근거 제공 |
| 상병 집중도 관리 | KCD 상병 코드 빈도 분석으로 특정 상병군 집중 리스크 조기 감지 |
| 거절 사유 패턴 파악 | 거절메모 키워드 분석으로 주요 거절 사유 데이터 기반 확인 |

##### 3.3.1 카테고리별 피처 목록

| D. 담보군 패턴 | D01~D02 | 주요담보집중도, 담보군다양도 |

#### 3.4 5차원 위험 스코어링 설계

##### 3.4.1 차원별 구성 및 가중치

| 차원 | 전체 가중치 | 구성 지표 및 세부 가중치 |
|---|---|---|
| DIM1 근접사고위험 | 25% | 근접건수(30%) + 근접비율(30%) + 최소경과일역순(25%) + 7일이내건수(15%) |
| DIM2 유형집중위험 | 18% | 유형집중도(35%) + 단일유형100%(35점 고정) + 담보집중도(20%) + 다발피보험자(10%) |
| DIM3 부정사기위험 | 25% | 사기의심건수(30%) + 사기의심비율(20%) + 사고발생후가입(20%) + 가입전사고(10%) + 텍스트사기징후(10%) + 내용짧음(10%) |
| DIM4 고손해볼륨위험 | 17% | 사고건수(25%) + 보험금합(30%) + 건당보험금(20%) + 다발피보험자비율(15%) + 고위험직군(10%) |
| DIM5 이력행태위험 | 15% | 현재유의(30점 고정) + 이력유의(15점 고정) + 유의피보험자(15%) + 설계사본인청구(20%) + 해약비율(10%) + 면책비율(10%) |

> DIM2·DIM5의 고정 점수 항목(단일유형100%, 현재유의, 이력유의)은 조건 충족 시 지정 점수를 직접 가산한다.
> 나머지 항목은 백분위 기반 표준화(rank(pct=True) × 100) 후 세부 가중치 적용.
> 전체 가중치 합계: 25 + 18 + 25 + 17 + 15 = 100%.

##### 3.4.2 스코어 산출 로직

```
[1단계] 피처 산출
  - 15개 카테고리(A~O) 기준 취급자코드 단위 집계

[2단계] 차원별 표준화
  - 백분위 기반: rank(pct=True) × 100 (취급자 전체 모집단 기준)
  - 고정 점수 항목은 조건 충족 여부로 직접 점수 부여

[3단계] 차원 점수 산출
  - 각 차원 내 구성 지표 × 세부 가중치 합산 → 차원 점수 (0~100)

[4단계] 종합 점수 산출
  - risk_score = DIM1×0.25 + DIM2×0.18 + DIM3×0.25 + DIM4×0.17 + DIM5×0.15
  - 최종 범위: 0~100 (높을수록 고위험)

[5단계] 등급 분류
  - 절대값 기준 4단계 분류 적용
```

#### 3.5 등급 분류 체계 (절대값 기준)

| 등급 | 점수 기준 | 권고조치 |
|---|---|---|
| CRITICAL | 65점 이상 | 즉시 설계 전면 제어 + 심사강화 |
| HIGH | 48~64점 | 신계약 심사강화 + 모니터링 강화 |
| MED | 32~47점 | 정기 모니터링 + 이상 시 알림 |
| LOW | 32점 미만 | 정상 관리 |

> 등급 기준은 절대값 기반으로 고정하며, 분위수 기반 분류를 병행 검토하지 않는다.

#### 3.6 손해 증가 패턴 감지 (4개 분석축)

| 분석축 | 집계 단위 | 분석 내용 |
|---|---|---|
| 1. 사고유형별 월별 손해 추이 | 사고유형 × 월 | 최근 6개월 vs 이전 6개월 건수/보험금 증가율 비교 |
| 2. 대리점별 손해 증가 패턴 | 대리점코드 | 대리점코드 단위 보험금 증가율 상위 구간 분석 |
| 3. 담보군 × 사고유형 조합 이상패턴 | 담보군 × 사고유형 조합 | 10건 이상 조합 기준 보험금합·평균보험금 분석 |
| 4. 지역별 손해 집중도 | 지역구 | 지역구 단위 건수·보험금·설계사수·건당보험금 분석 |

> 분석축 1의 비교 기간(최근 6개월 vs 이전 6개월)은 기준일 기준 rolling 적용.
> 분석축 3의 최소 건수 기준(10건)은 실데이터 규모 확인 후 재검토 가능.

#### 3.7 출력 산출물 (6개 CSV)

| 순번 | 파일명 | 주요 내용 |
|---|---|---|
| 1 | 취급자위험모델_결과.csv | 주요 피처 + 5차원 점수 + 종합점수 + 등급 + 권고조치 |
| 2 | 사고유형_손해증가트렌드.csv | 사고유형별 최근·이전 6개월 비교 증가율 |
| 3 | 대리점_손해증가트렌드.csv | 대리점코드별 보험금 증가율 상위 분석 |
| 4 | 담보유형조합_손해분석.csv | 담보군 × 사고유형 조합 보험금합·평균보험금 |
| 5 | 지역별_손해분석.csv | 지역구 단위 건수·보험금·설계사수·건당보험금 |
| 6 | 취급자위험모델_피처전체.csv | 15개 카테고리 전체 피처값 원본 |
