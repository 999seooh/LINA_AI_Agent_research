## M-01 손해율 분석 모델

### 1. 목적

#### 1.1 분석 목적

기간별·상품별 손해율 추이를 집계하고, 임계값 초과 구간에 대한 이상 감지 로직을 적용하여 UW 팀의 조기 경보 및 포트폴리오 리스크 관리를 지원한다.

#### 1.2 모델 적용 범위

| 항목 | 내용 |
|---|---|
| 분석 단위 | 상품코드 × 계약연월 |
| 집계 기간 | 월별 추이 최소 12개월 (실데이터 제공 시 확정) |
| 주요 분석 관점 | 손해율 추이, 이상 감지, 채널별 분리 |
| 기대 효과 | 손해율 악화 조기 감지, 상품별 리스크 차별화 관리 |

#### 1.3 활용 컬럼

| 컬럼명 | 역할 |
|---|---|
| contract_ym | 집계 기준 기간 |
| product_code | 상품별 분리 집계 기준 |
| product_group | 상품군 상위 분류 |
| earned_premium | 손해율 분모 |
| incurred_loss | 손해율 분자 |
| loss_ratio | 파생 지표 또는 원천 제공 |
| claim_count | 이상 감지 보조 지표 |
| channel_code | 채널별 분리 분석 |

### 3. 변경/설계안

#### 3.1 모델 유형

통계 기반 집계 + 이상 감지 (Statistical Aggregation + Anomaly Detection)

#### 3.2 집계 기준

| 항목 | 기준 |
|---|---|
| 기준일 | 계약연월(contract_ym) 기준 월말 마감 (실데이터 제공 시 확정) |
| 집계 단위 | product_code × contract_ym (상품코드 × 계약연월) |
| 상위 집계 | product_group 단위 롤업 집계 병행 |
| 채널 분리 | channel_code 기준 서브 집계 |
| 필터 조건 | earned_premium > 0인 유효 계약 기준 (실데이터 제공 시 확정) |

#### 3.3 핵심 지표 정의

| 지표명 | 산식 | 설명 |
|---|---|---|
| 손해율 (Loss Ratio) | incurred_loss / earned_premium × 100 | 핵심 KPI. 원천 제공 시 원천 우선 사용 |
| 손해율 전월 대비 변화율 | (당월 LR - 전월 LR) / 전월 LR × 100 | 추이 이상 감지 보조 |
| 청구 빈도 | claim_count / 계약건수 | 청구 집중도 모니터링 |
| 3개월 이동평균 손해율 | 최근 3개월 손해율 단순 평균 | 노이즈 제거 후 추세 확인 |
| 12개월 누적 손해율 | 12개월 누적 incurred_loss / 누적 earned_premium | 연간 기준 포트폴리오 성과 |

#### 3.4 이상 감지 로직

**임계값 기반 이상 감지 (Rule-based Alert)**

| 감지 조건 | 기준값 | 처리 방법 |
|---|---|---|
| 단일 월 손해율 초과 | TBD% (실데이터 기반 확정 필요) | 해당 product_code 플래그 처리 |
| 전월 대비 손해율 급등 | +15%p 초과 (검토 필요) | 변화율 이상 플래그 |
| 3개월 연속 손해율 상승 | 3개월 연속 증가 | 추세 이상 플래그 |
| 청구 건수 이상 급증 | 전월 대비 +30% 초과 (검토 필요) | 청구 이상 플래그 |

> 임계값은 모두 "검토 필요" 상태로, 실데이터 분포 확인 후 최종 확정한다.

**통계 기반 이상 감지 (Statistical Outlier)**

| 방법 | 기준 | 설명 |
|---|---|---|
| Z-score | \|Z\| > 2.5 (검토 필요) | 상품 × 월 단위 표준화 후 이상치 판별 |
| IQR 방법 | Q3 + 1.5×IQR 초과 | 손해율 분포 상단 이상치 탐지 |

#### 3.5 집계 흐름

```
[원천 데이터]
  → 유효 계약 필터 (earned_premium > 0)
  → product_code × contract_ym 단위 집계
  → 손해율 산출 (파생 또는 원천)
  → 이동평균·변화율 파생 지표 산출
  → 임계값 비교 → 이상 플래그 생성
  → 상품군(product_group) 롤업
  → 채널(channel_code) 서브 집계
  → 최종 출력 테이블
```

### 4. 테스트 기준

| 테스트 항목 | 기준값 | 검증 방법 | 담당 |
|---|---|---|---|
| 손해율 산출 정확성 | incurred_loss / earned_premium 수식 일치 | 샘플 5건 수동 계산 대비 | 검토 필요 |
| 집계 단위 누락 여부 | product_code 전체 집계 완전성 100% | 원천 대비 집계 건수 비교 | 검토 필요 |
| 이상 감지 플래그 정확성 | 임계값 초과 건 플래그 누락률 0% | 조건 수동 검증 | 검토 필요 |
| 이동평균 산출 | 3개월 이동평균 수식 일치 | 구간별 수동 검증 | 검토 필요 |
| 채널 분리 집계 | channel_code 기준 합산 = 전체 합산 | 합계 일치 검증 | 검토 필요 |
| 이상치 Z-score | 임계 이상 건 누락 없음 | 전체 분포 확인 | 검토 필요 |

## M-02 유지율 분석 모델

### 1. 목적

#### 1.1 분석 목적

계약 코호트 기준의 유지율·해지율 패턴을 분석하고, 경과 월수별 생존율 변화를 추적하여 조기 이탈 구간 식별 및 채널·상품별 유지 성과를 비교한다.

#### 1.2 모델 적용 범위

| 항목 | 내용 |
|---|---|
| 분석 단위 | 계약 코호트 (가입월 기준) × 경과 월수 |
| 집계 기간 | 실데이터 제공 시 확정 |
| 주요 분석 관점 | 코호트별 생존율, 해지 사유 분포, 채널·상품별 유지율 비교 |
| 기대 효과 | 조기 해지 구간 식별, 유지율 취약 채널·상품 관리 |

#### 1.3 활용 컬럼

| 컬럼명 | 역할 |
|---|---|
| contract_date | 코호트 기준 (가입월 추출) |
| lapse_date | 해지 시점 산출 |
| lapse_yn | 해지 여부 분류 |
| lapse_reason_code | 해지 사유별 분포 분석 |
| policy_duration_month | 생존 분석 경과 기간 |
| product_code | 상품별 유지율 분리 |
| age_band | 연령대별 유지율 분리 |
| channel_code | 채널별 유지율 분리 |
| uw_result_code | 인수 조건별 유지율 연계 분석 |

신규 설계 (현행 모델 없음)

### 3. 변경/설계안

#### 3.1 모델 유형

코호트 분석 + 생존 분석 (Cohort Analysis + Survival Analysis)

#### 3.2 집계 기준

| 항목 | 기준 |
|---|---|
| 기준일 | 분석 기준일 (실데이터 제공 시 확정) |
| 코호트 기준 | contract_date 기준 가입연월 (YYYYMM) |
| 경과 기간 단위 | policy_duration_month (1개월 단위) |
| 생존 정의 | lapse_yn = 'N' (유지 중) |
| 이탈 정의 | lapse_yn = 'Y' (해지 완료) |
| 필터 조건 | 유효 계약 기준 (이상 계약 제외 조건 실데이터 제공 시 확정) |

#### 3.3 핵심 지표 정의

| 지표명 | 산식 | 설명 |
|---|---|---|
| N개월 유지율 | 해당 코호트 내 N개월 유지 계약수 / 최초 계약수 × 100 | 코호트별 시점 유지율 |
| 해지율 (기간별) | 해당 기간 내 신규 해지수 / 직전 기간 유지수 × 100 | 기간별 이탈 속도 |
| 조기 해지율 | 가입 후 12개월 이내 해지율 | 불완전 판매 리스크 지표 |
| 해지 사유별 비율 | 사유별 해지수 / 전체 해지수 × 100 | lapse_reason_code 기반 |
| 채널별 유지율 차이 | 채널 A 유지율 - 채널 B 유지율 | 채널 간 유지 성과 비교 |
| UW 조건별 유지율 | uw_result_code 기준 유지율 분리 집계 | 인수 조건과 유지 성과 연계 |

#### 3.4 코호트 분석 설계

**코호트 매트릭스 구조**

- 행(Row): 가입 코호트 (계약연월)
- 열(Column): 경과 월수 (1M, 3M, 6M, 12M, 24M, 36M 등)
- 셀(Cell): 해당 코호트 × 경과 시점 유지율(%)

**분석 차원**

| 차원 | 분리 기준 | 목적 |
|---|---|---|
| 전체 | 전체 계약 | 기준선 유지율 |
| 상품별 | product_code | 상품 간 유지 성과 비교 |
| 채널별 | channel_code | 채널 유지율 취약 구간 식별 |
| 연령대별 | age_band | 연령대별 이탈 패턴 확인 |
| UW 조건별 | uw_result_code | 인수 조건 영향 분석 |

#### 3.5 생존 분석 적용 (검토 필요)

> 생존 분석(Kaplan-Meier 추정) 적용 여부는 실데이터 규모 및 활용 목적 확인 후 결정한다.

| 항목 | 내용 |
|---|---|
| 적용 방법 | Kaplan-Meier 생존함수 추정 |
| 이벤트 정의 | 해지 (lapse_yn = 'Y') |
| 중도절단(Censoring) 처리 | 분석 기준일 기준 미해지 계약 |
| 비교 그룹 | 채널별, 상품별, uw_result_code별 생존 곡선 비교 |
| 검정 방법 | Log-rank test (그룹 간 차이 유의성 검토 필요) |

#### 3.6 해지 사유 분류 체계

| lapse_reason_code 유형 | 분류명 | 리스크 의미 |
|---|---|---|
| 자발적 해지 | 자발적 이탈 | 상품 불만족, 경쟁사 전환 |
| 미납 해지 | 미납 이탈 | 재정 악화, 유지 능력 저하 |
| 직권 해지 | 직권 이탈 | UW 재심사 또는 계약 무효 |
| 기타 | 기타 | 실데이터 제공 시 세부 분류 확정 |

| 테스트 항목 | 기준값 | 검증 방법 | 담당 |

| 중도절단 처리 정확성 | 미해지 계약 제외 오류 0% | NULL lapse_date 건 확인 | 검토 필요 |
| 채널 분리 완전성 | channel_code 전체 커버리지 100% | 원천 채널 목록 대비 확인 | 검토 필요 |
| 해지 사유 분류 | lapse_reason_code 미분류 건 = 0 | 코드 목록 전수 확인 | 검토 필요 |

| 상품 포트폴리오 최적화 | 유지율 취약 상품 조기 식별 및 인수 기준 재검토 트리거 |

## M-03 심사데이터 분석 모델

### 1. 목적

#### 1.1 분석 목적

UW 심사 결과 분포, 심사 소요일 효율, 결과별 후행 손해율을 집계·분석하여 심사 품질 모니터링 및 인수 기준 고도화 근거를 제공한다.

#### 1.2 모델 적용 범위

| 항목 | 내용 |
|---|---|
| 분석 단위 | 심사일 × 상품코드 |
| 결과 분류 | 승인 / 조건부 승인 / 거절 / 보류 |
| 주요 분석 관점 | 결과 분포, 소요일 효율, 위험 변수별 결과 패턴 |
| 기대 효과 | 심사 효율 개선, 인수 기준 데이터 기반 검토 |

#### 1.3 활용 컬럼

| 컬럼명 | 역할 |
|---|---|
| uw_date | 집계 기준일 |
| uw_result_code | 결과 분포 집계 기준 |
| uw_condition_code | 조건부 승인 세부 유형 분류 |
| uw_processing_days | 심사 효율 지표 |
| uw_channel | 채널별 심사 패턴 분석 |

## M-04 보험취급자 위험 관리 모델

### 1. 목적

#### 1.1 분석 목적

설계사(취급자)별 복수 위험 변수를 기반으로 다차원 위험도 스코어링을 산출하고, 등급 분류 체계를 통해 고위험 취급자를 조기 식별한다. 손해 증가 유형 및 패턴을 감지하여 불완전 판매·역선택·부정 청구 관련 리스크를 관리한다.

#### 1.2 모델 적용 범위

| 항목 | 내용 |
|---|---|
| 분석 단위 | 설계사 코드 (agent_code) 단위 |
| 상위 집계 | 대리점 코드 (agency_code) 단위 롤업 병행 |
| 주요 분석 관점 | 다차원 위험도 스코어링, 등급 분류, 손해 증가 패턴 감지 |
| 기대 효과 | 고위험 취급자 조기 식별, 역선택·부정 청구 리스크 감소 |

### 3. 변경/설계안

#### 3.1 모델 유형

다차원 스코어링 — 룰+통계 하이브리드 (Multi-dimensional Scoring: Rule + Statistical Hybrid)

#### 3.2 집계 기준

| 항목 | 기준 |
|---|---|
| 기준일 | 분석 기준월 말일 (실데이터 제공 시 확정) |
| 집계 단위 | agent_code 단위 월별 집계 |
| 상위 집계 | agency_code 단위 롤업 (설계사 집합 위험도 산출) |
| 최소 활동량 기준 | contract_count >= N건 이상 (N은 실데이터 기반 확정, 소규모 취급자 노이즈 제거 목적) |
| 분석 기간 | 최근 12개월 rolling 기준 (검토 필요) |

#### 3.3 다차원 위험도 스코어링 설계

##### 3.3.1 스코어링 변수 및 가중치 체계

| 변수명 | 컬럼 | 가중치(안) | 측정 기준 | 높을수록 위험 |
|---|---|---|---|---|
| 손해율 수준 | loss_ratio | 25% | 취급 계약 손해율 절대값 | 예 |
| 손해율 증가 추이 | loss_ratio_trend | 20% | 최근 N개월 변화율 | 예 |
| 청구 집중도 | claim_count / contract_count | 15% | 계약 대비 청구 비율 | 예 |
| 조기 해지율 | early_lapse_rate | 15% | 12개월 이내 해지율 | 예 |
| 병원 집중도 | hospital_concentration | 10% | 특정 병원 청구 집중도 | 예 |
| 고액 계약 비율 | high_sum_insured_count / contract_count | 8% | 고액 계약 편중도 | 예 |
| 상품 편중도 | product_concentration | 7% | 특정 상품 집중도 | 예 (극단적 집중 시) |

> 가중치는 초안으로, 실데이터 분포 확인 및 전문가 검토 후 최종 확정한다.
> 각 변수는 표준화(Z-score 또는 Min-Max) 후 가중 합산한다.

##### 3.3.2 스코어 산출 로직

```
[1단계] 변수별 원값 산출
  - 각 agent_code 단위로 집계 기간 내 변수값 산출

[2단계] 변수 표준화
  - 각 변수: Min-Max 정규화 (0~1 스케일) 또는 Z-score (실데이터 분포 확인 후 결정)
  - 이상치 처리: 상하위 1% 윈저화(Winsorization) 적용 (검토 필요)

[3단계] 가중 합산
  - risk_score = SUM(표준화값_i × 가중치_i) × 100
  - 최종 스코어 범위: 0~100 (높을수록 고위험)

[4단계] 등급 분류
  - risk_score 기준으로 risk_grade 분류

[5단계] 룰 기반 Override 적용
  - 개별 변수가 하드 임계값 초과 시 등급 강제 상향
```

##### 3.3.3 등급 분류 체계

| 등급 | 위험 수준 | risk_score 범위(안) | 관리 방침 |
|---|---|---|---|
| S | 매우 높음 (긴급 관리) | 80점 이상 | 즉시 보고 및 집중 모니터링 |
| A | 높음 (주의 관찰) | 60~79점 | 정기 모니터링 강화 |
| B | 보통 (관찰 유지) | 40~59점 | 정기 모니터링 |
| C | 낮음 (표준 관리) | 20~39점 | 일반 관리 |
| D | 매우 낮음 (우수) | 20점 미만 | 관리 불요 |

> 등급 구간 경계값은 실데이터 분포 확인 및 전문가 검토 후 최종 확정한다.
> 초기에는 분위수(Percentile) 기반 등급 분류를 병행 검토한다.

#### 3.4 손해 증가 유형/패턴 감지 로직

##### 3.4.1 패턴 유형 분류

| 패턴 유형 | 감지 기준 | 활용 컬럼 | 위험 해석 |
|---|---|---|---|
| 급격한 손해율 증가 | loss_ratio_trend > +20%p (N개월 내, 검토 필요) | loss_ratio, loss_ratio_trend | 특정 시점 계약 집중 청구 |
| 특정 병원 집중 | hospital_concentration > X% (검토 필요) | hospital_code, hospital_concentration | 병원-설계사 담합 리스크 |
| 조기 해지 급증 | early_lapse_rate > Y% (검토 필요) | early_lapse_count, early_lapse_rate | 불완전 판매 가능성 |
| 고액 계약 편중 | high_sum_insured_count / contract_count > Z% (검토 필요) | high_sum_insured_count | 역선택 유도 가능성 |
| 특정 사고 유형 집중 | accident_type_code 단일 유형 비율 > W% (검토 필요) | accident_type_code | 허위/과다 청구 패턴 |
| 상품 극단 편중 | product_concentration > V% (검토 필요) | product_concentration | 특정 상품 역선택 유도 |

> 모든 임계값(X, Y, Z, W, V)은 "검토 필요" 상태로, 실데이터 분포 확인 후 확정한다.

##### 3.4.2 사고 내용 텍스트 패턴 감지 (검토 필요)

> accident_description 컬럼은 개인식별 표현 제거 후 사용. 활용 방식은 실데이터 제공 시 확정한다.

| 감지 방법 | 내용 | 적용 조건 |
|---|---|---|
| 키워드 기반 룰 | 사고 내용 내 반복 키워드 추출 | 개인식별 제거 완료 후 |
| 유사 패턴 군집 | 동일/유사 사고 내용 반복 패턴 감지 | 실데이터 규모 확인 후 결정 |
| 빈도 이상 탐지 | 동일 사고 유형 반복 청구 집중 | accident_type_code 기반 선행 탐지 가능 |

##### 3.4.3 대리점 단위 위험 롤업

| 항목 | 기준 |
|---|---|
| 집계 단위 | agency_code 기준 소속 설계사 스코어 집계 |
| 집계 방법 | 소속 설계사 중 S등급·A등급 비율 산출 |
| 대리점 위험 지표 | S+A 비율이 전체 대리점 상위 분위 초과 시 고위험 대리점 플래그 |

#### 3.5 룰 기반 Override 체계

스코어와 무관하게 아래 단일 조건 충족 시 등급 강제 상향한다.

| Override 조건 | 강제 등급 | 근거 |
|---|---|---|
| loss_ratio > 하드 임계값 (실데이터 기반 확정) | 최소 A등급 | 손해율 절대값 위험 |
| hospital_concentration 극단값 초과 (검토 필요) | 최소 A등급 | 병원 집중 위험 |
| early_lapse_rate 극단값 초과 (검토 필요) | 최소 B등급 | 불완전 판매 위험 |
| 특정 사고 유형 단독 집중 극단값 (검토 필요) | 최소 A등급 | 부정 청구 패턴 위험 |
