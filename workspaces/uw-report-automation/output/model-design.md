## M-01 손해율 분석 모델

### 1. 목적

#### 1.1 분석 목적

기간별·상품별 손해율 추이를 집계하고, 임계값 초과 구간에 대한 이상 감지 로직을 적용하여 UW 팀의 조기 경보 및 포트폴리오 리스크 관리를 지원한다.

#### 1.2 모델 적용 범위

| 항목 | 내용 |
|---|---|
| 분석 단위 | 상품코드 × 계약연월 |
| 집계 기간 | 월별 추이 최소 12개월 (실데이터 제공 시 확정) |
| 주요 분석 관점 | 손해율 추이, 이상 감지, 채널별 분리 |
| 기대 효과 | 손해율 악화 조기 감지, 상품별 리스크 차별화 관리 |

#### 1.3 활용 컬럼

| 컬럼명 | 역할 |
|---|---|
| contract_ym | 집계 기준 기간 |
| product_code | 상품별 분리 집계 기준 |
| product_group | 상품군 상위 분류 |
| earned_premium | 손해율 분모 |
| incurred_loss | 손해율 분자 |
| loss_ratio | 파생 지표 또는 원천 제공 |
| claim_count | 이상 감지 보조 지표 |
| channel_code | 채널별 분리 분석 |

### 3. 변경/설계안

#### 3.1 모델 유형

통계 기반 집계 + 이상 감지 (Statistical Aggregation + Anomaly Detection)

#### 3.2 집계 기준

| 항목 | 기준 |
|---|---|
| 기준일 | 계약연월(contract_ym) 기준 월말 마감 (실데이터 제공 시 확정) |
| 집계 단위 | product_code × contract_ym (상품코드 × 계약연월) |
| 상위 집계 | product_group 단위 롤업 집계 병행 |
| 채널 분리 | channel_code 기준 서브 집계 |
| 필터 조건 | earned_premium > 0인 유효 계약 기준 (실데이터 제공 시 확정) |

#### 3.3 핵심 지표 정의

| 지표명 | 산식 | 설명 |
|---|---|---|
| 손해율 (Loss Ratio) | incurred_loss / earned_premium × 100 | 핵심 KPI. 원천 제공 시 원천 우선 사용 |
| 손해율 전월 대비 변화율 | (당월 LR - 전월 LR) / 전월 LR × 100 | 추이 이상 감지 보조 |
| 청구 빈도 | claim_count / 계약건수 | 청구 집중도 모니터링 |
| 3개월 이동평균 손해율 | 최근 3개월 손해율 단순 평균 | 노이즈 제거 후 추세 확인 |
| 12개월 누적 손해율 | 12개월 누적 incurred_loss / 누적 earned_premium | 연간 기준 포트폴리오 성과 |

#### 3.4 이상 감지 로직

**임계값 기반 이상 감지 (Rule-based Alert)**

| 감지 조건 | 기준값 | 처리 방법 |
|---|---|---|
| 단일 월 손해율 초과 | TBD% (실데이터 기반 확정 필요) | 해당 product_code 플래그 처리 |
| 전월 대비 손해율 급등 | +15%p 초과 (검토 필요) | 변화율 이상 플래그 |
| 3개월 연속 손해율 상승 | 3개월 연속 증가 | 추세 이상 플래그 |
| 청구 건수 이상 급증 | 전월 대비 +30% 초과 (검토 필요) | 청구 이상 플래그 |

> 임계값은 모두 "검토 필요" 상태로, 실데이터 분포 확인 후 최종 확정한다.

**통계 기반 이상 감지 (Statistical Outlier)**

| 방법 | 기준 | 설명 |
|---|---|---|
| Z-score | \|Z\| > 2.5 (검토 필요) | 상품 × 월 단위 표준화 후 이상치 판별 |
| IQR 방법 | Q3 + 1.5×IQR 초과 | 손해율 분포 상단 이상치 탐지 |

#### 3.5 집계 흐름

```
[원천 데이터]
  → 유효 계약 필터 (earned_premium > 0)
  → product_code × contract_ym 단위 집계
  → 손해율 산출 (파생 또는 원천)
  → 이동평균·변화율 파생 지표 산출
  → 임계값 비교 → 이상 플래그 생성
  → 상품군(product_group) 롤업
  → 채널(channel_code) 서브 집계
  → 최종 출력 테이블
```

### 4. 테스트 기준

| 테스트 항목 | 기준값 | 검증 방법 | 담당 |
|---|---|---|---|
| 손해율 산출 정확성 | incurred_loss / earned_premium 수식 일치 | 샘플 5건 수동 계산 대비 | 검토 필요 |
| 집계 단위 누락 여부 | product_code 전체 집계 완전성 100% | 원천 대비 집계 건수 비교 | 검토 필요 |
| 이상 감지 플래그 정확성 | 임계값 초과 건 플래그 누락률 0% | 조건 수동 검증 | 검토 필요 |
| 이동평균 산출 | 3개월 이동평균 수식 일치 | 구간별 수동 검증 | 검토 필요 |
| 채널 분리 집계 | channel_code 기준 합산 = 전체 합산 | 합계 일치 검증 | 검토 필요 |
| 이상치 Z-score | 임계 이상 건 누락 없음 | 전체 분포 확인 | 검토 필요 |

## M-02 유지율 분석 모델

### 1. 목적

#### 1.1 분석 목적

계약 코호트 기준의 유지율·해지율 패턴을 분석하고, 경과 월수별 생존율 변화를 추적하여 조기 이탈 구간 식별 및 채널·상품별 유지 성과를 비교한다.

#### 1.2 모델 적용 범위

| 항목 | 내용 |
|---|---|
| 분석 단위 | 계약 코호트 (가입월 기준) × 경과 월수 |
| 집계 기간 | 실데이터 제공 시 확정 |
| 주요 분석 관점 | 코호트별 생존율, 해지 사유 분포, 채널·상품별 유지율 비교 |
| 기대 효과 | 조기 해지 구간 식별, 유지율 취약 채널·상품 관리 |

#### 1.3 활용 컬럼

| 컬럼명 | 역할 |
|---|---|
| contract_date | 코호트 기준 (가입월 추출) |
| lapse_date | 해지 시점 산출 |
| lapse_yn | 해지 여부 분류 |
| lapse_reason_code | 해지 사유별 분포 분석 |
| policy_duration_month | 생존 분석 경과 기간 |
| product_code | 상품별 유지율 분리 |
| age_band | 연령대별 유지율 분리 |
| channel_code | 채널별 유지율 분리 |
| uw_result_code | 인수 조건별 유지율 연계 분석 |

신규 설계 (현행 모델 없음)

### 3. 변경/설계안

#### 3.1 모델 유형

코호트 분석 + 생존 분석 (Cohort Analysis + Survival Analysis)

#### 3.2 집계 기준

| 항목 | 기준 |
|---|---|
| 기준일 | 분석 기준일 (실데이터 제공 시 확정) |
| 코호트 기준 | contract_date 기준 가입연월 (YYYYMM) |
| 경과 기간 단위 | policy_duration_month (1개월 단위) |
| 생존 정의 | lapse_yn = 'N' (유지 중) |
| 이탈 정의 | lapse_yn = 'Y' (해지 완료) |
| 필터 조건 | 유효 계약 기준 (이상 계약 제외 조건 실데이터 제공 시 확정) |

#### 3.3 핵심 지표 정의

| 지표명 | 산식 | 설명 |
|---|---|---|
| N개월 유지율 | 해당 코호트 내 N개월 유지 계약수 / 최초 계약수 × 100 | 코호트별 시점 유지율 |
| 해지율 (기간별) | 해당 기간 내 신규 해지수 / 직전 기간 유지수 × 100 | 기간별 이탈 속도 |
| 조기 해지율 | 가입 후 12개월 이내 해지율 | 불완전 판매 리스크 지표 |
| 해지 사유별 비율 | 사유별 해지수 / 전체 해지수 × 100 | lapse_reason_code 기반 |
| 채널별 유지율 차이 | 채널 A 유지율 - 채널 B 유지율 | 채널 간 유지 성과 비교 |
| UW 조건별 유지율 | uw_result_code 기준 유지율 분리 집계 | 인수 조건과 유지 성과 연계 |

#### 3.4 코호트 분석 설계

**코호트 매트릭스 구조**

- 행(Row): 가입 코호트 (계약연월)
- 열(Column): 경과 월수 (1M, 3M, 6M, 12M, 24M, 36M 등)
- 셀(Cell): 해당 코호트 × 경과 시점 유지율(%)

**분석 차원**

| 차원 | 분리 기준 | 목적 |
|---|---|---|
| 전체 | 전체 계약 | 기준선 유지율 |
| 상품별 | product_code | 상품 간 유지 성과 비교 |
| 채널별 | channel_code | 채널 유지율 취약 구간 식별 |
| 연령대별 | age_band | 연령대별 이탈 패턴 확인 |
| UW 조건별 | uw_result_code | 인수 조건 영향 분석 |

#### 3.5 생존 분석 적용 (검토 필요)

> 생존 분석(Kaplan-Meier 추정) 적용 여부는 실데이터 규모 및 활용 목적 확인 후 결정한다.

| 항목 | 내용 |
|---|---|
| 적용 방법 | Kaplan-Meier 생존함수 추정 |
| 이벤트 정의 | 해지 (lapse_yn = 'Y') |
| 중도절단(Censoring) 처리 | 분석 기준일 기준 미해지 계약 |
| 비교 그룹 | 채널별, 상품별, uw_result_code별 생존 곡선 비교 |
| 검정 방법 | Log-rank test (그룹 간 차이 유의성 검토 필요) |

#### 3.6 해지 사유 분류 체계

| lapse_reason_code 유형 | 분류명 | 리스크 의미 |
|---|---|---|
| 자발적 해지 | 자발적 이탈 | 상품 불만족, 경쟁사 전환 |
| 미납 해지 | 미납 이탈 | 재정 악화, 유지 능력 저하 |
| 직권 해지 | 직권 이탈 | UW 재심사 또는 계약 무효 |
| 기타 | 기타 | 실데이터 제공 시 세부 분류 확정 |

| 테스트 항목 | 기준값 | 검증 방법 | 담당 |

| 중도절단 처리 정확성 | 미해지 계약 제외 오류 0% | NULL lapse_date 건 확인 | 검토 필요 |
| 채널 분리 완전성 | channel_code 전체 커버리지 100% | 원천 채널 목록 대비 확인 | 검토 필요 |
| 해지 사유 분류 | lapse_reason_code 미분류 건 = 0 | 코드 목록 전수 확인 | 검토 필요 |

| 상품 포트폴리오 최적화 | 유지율 취약 상품 조기 식별 및 인수 기준 재검토 트리거 |

## M-03 심사데이터 분석 모델

### 1. 목적

#### 1.1 분석 목적

UW 심사 결과 분포, 심사 소요일 효율, 결과별 후행 손해율을 집계·분석하여 심사 품질 모니터링 및 인수 기준 고도화 근거를 제공한다.

#### 1.2 모델 적용 범위

| 항목 | 내용 |
|---|---|
| 분석 단위 | 심사일 × 상품코드 |
| 결과 분류 | 승인 / 조건부 승인 / 거절 / 보류 |
| 주요 분석 관점 | 결과 분포, 소요일 효율, 위험 변수별 결과 패턴 |
| 기대 효과 | 심사 효율 개선, 인수 기준 데이터 기반 검토 |

#### 1.3 활용 컬럼

| 컬럼명 | 역할 |
|---|---|
| uw_date | 집계 기준일 |
| uw_result_code | 결과 분포 집계 기준 |
| uw_condition_code | 조건부 승인 세부 유형 분류 |
| uw_processing_days | 심사 효율 지표 |

#### 3.4 5차원 위험 스코어링 설계

##### 3.4.1 차원별 구성 및 가중치

| 차원 | 전체 가중치 | 구성 지표 및 세부 가중치 |
|---|---|---|
| DIM1 근접사고위험 | 25% | 근접건수(30%) + 근접비율(30%) + 최소경과일역순(25%) + 7일이내건수(15%) |
| DIM2 유형집중위험 | 18% | 유형집중도(35%) + 단일유형100%(35점 고정) + 담보집중도(20%) + 다발피보험자(10%) |
| DIM3 부정사기위험 | 25% | 사기의심건수(30%) + 사기의심비율(20%) + 사고발생후가입(20%) + 가입전사고(10%) + 텍스트사기징후(10%) + 내용짧음(10%) |
| DIM4 고손해볼륨위험 | 17% | 사고건수(25%) + 보험금합(30%) + 건당보험금(20%) + 다발피보험자비율(15%) + 고위험직군(10%) |
| DIM5 이력행태위험 | 15% | 현재유의(30점 고정) + 이력유의(15점 고정) + 유의피보험자(15%) + 설계사본인청구(20%) + 해약비율(10%) + 면책비율(10%) |

> DIM2·DIM5의 고정 점수 항목(단일유형100%, 현재유의, 이력유의)은 조건 충족 시 지정 점수를 직접 가산한다.
> 나머지 항목은 백분위 기반 표준화(rank(pct=True) × 100) 후 세부 가중치 적용.
> 전체 가중치 합계: 25 + 18 + 25 + 17 + 15 = 100%.

##### 3.4.2 스코어 산출 로직

```
[1단계] 피처 산출
  - 15개 카테고리(A~O) 기준 취급자코드 단위 집계

[2단계] 차원별 표준화
  - 백분위 기반: rank(pct=True) × 100 (취급자 전체 모집단 기준)
  - 고정 점수 항목은 조건 충족 여부로 직접 점수 부여

[3단계] 차원 점수 산출
  - 각 차원 내 구성 지표 × 세부 가중치 합산 → 차원 점수 (0~100)

[4단계] 종합 점수 산출
  - risk_score = DIM1×0.25 + DIM2×0.18 + DIM3×0.25 + DIM4×0.17 + DIM5×0.15
  - 최종 범위: 0~100 (높을수록 고위험)

[5단계] 등급 분류
  - 절대값 기준 4단계 분류 적용
```

#### 3.5 등급 분류 체계 (절대값 기준)

| 등급 | 점수 기준 | 권고조치 |
|---|---|---|
| CRITICAL | 65점 이상 | 즉시 설계 전면 제어 + 심사강화 |
| HIGH | 48~64점 | 신계약 심사강화 + 모니터링 강화 |
| MED | 32~47점 | 정기 모니터링 + 이상 시 알림 |
| LOW | 32점 미만 | 정상 관리 |

> 등급 기준은 절대값 기반으로 고정하며, 분위수 기반 분류를 병행 검토하지 않는다.

#### 3.6 손해 증가 패턴 감지 (4개 분석축)

| 분석축 | 집계 단위 | 분석 내용 |
|---|---|---|
| 1. 사고유형별 월별 손해 추이 | 사고유형 × 월 | 최근 6개월 vs 이전 6개월 건수/보험금 증가율 비교 |
| 2. 대리점별 손해 증가 패턴 | 대리점코드 | 대리점코드 단위 보험금 증가율 상위 구간 분석 |
| 3. 담보군 × 사고유형 조합 이상패턴 | 담보군 × 사고유형 조합 | 10건 이상 조합 기준 보험금합·평균보험금 분석 |

> 분석축 1의 비교 기간(최근 6개월 vs 이전 6개월)은 기준일 기준 rolling 적용.
> 분석축 3의 최소 건수 기준(10건)은 실데이터 규모 확인 후 재검토 가능.
