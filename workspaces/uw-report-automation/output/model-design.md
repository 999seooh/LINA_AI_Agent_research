## M-01 손해율 분석 모델

### 1. 목적

#### 1.1 분석 목적

기간별·상품별 손해율 추이를 집계하고, 임계값 초과 구간에 대한 이상 감지 로직을 적용하여 UW 팀의 조기 경보 및 포트폴리오 리스크 관리를 지원한다.

#### 1.2 모델 적용 범위

| 항목 | 내용 |
|---|---|
| 분석 단위 | 상품코드 × 계약연월 |
| 집계 기간 | 월별 추이 최소 12개월 (실데이터 제공 시 확정) |
| 주요 분석 관점 | 손해율 추이, 이상 감지, 채널별 분리 |
| 기대 효과 | 손해율 악화 조기 감지, 상품별 리스크 차별화 관리 |

#### 1.3 활용 컬럼

| 컬럼명 | 역할 |
|---|---|
| contract_ym | 집계 기준 기간 |
| product_code | 상품별 분리 집계 기준 |
| product_group | 상품군 상위 분류 |
| earned_premium | 손해율 분모 |
| incurred_loss | 손해율 분자 |
| loss_ratio | 파생 지표 또는 원천 제공 |
| claim_count | 이상 감지 보조 지표 |
| channel_code | 채널별 분리 분석 |

### 3. 변경/설계안

#### 3.1 모델 유형

통계 기반 집계 + 이상 감지 (Statistical Aggregation + Anomaly Detection)

#### 3.2 집계 기준

| 항목 | 기준 |
|---|---|
| 기준일 | 계약연월(contract_ym) 기준 월말 마감 (실데이터 제공 시 확정) |
| 집계 단위 | product_code × contract_ym (상품코드 × 계약연월) |
| 상위 집계 | product_group 단위 롤업 집계 병행 |
| 채널 분리 | channel_code 기준 서브 집계 |
| 필터 조건 | earned_premium > 0인 유효 계약 기준 (실데이터 제공 시 확정) |

#### 3.3 핵심 지표 정의

| 지표명 | 산식 | 설명 |
|---|---|---|
| 손해율 (Loss Ratio) | incurred_loss / earned_premium × 100 | 핵심 KPI. 원천 제공 시 원천 우선 사용 |
| 손해율 전월 대비 변화율 | (당월 LR - 전월 LR) / 전월 LR × 100 | 추이 이상 감지 보조 |
| 청구 빈도 | claim_count / 계약건수 | 청구 집중도 모니터링 |
| 3개월 이동평균 손해율 | 최근 3개월 손해율 단순 평균 | 노이즈 제거 후 추세 확인 |
| 12개월 누적 손해율 | 12개월 누적 incurred_loss / 누적 earned_premium | 연간 기준 포트폴리오 성과 |

#### 3.4 이상 감지 로직

**임계값 기반 이상 감지 (Rule-based Alert)**

| 감지 조건 | 기준값 | 처리 방법 |
|---|---|---|
| 단일 월 손해율 초과 | TBD% (실데이터 기반 확정 필요) | 해당 product_code 플래그 처리 |
| 전월 대비 손해율 급등 | +15%p 초과 (검토 필요) | 변화율 이상 플래그 |
| 3개월 연속 손해율 상승 | 3개월 연속 증가 | 추세 이상 플래그 |
| 청구 건수 이상 급증 | 전월 대비 +30% 초과 (검토 필요) | 청구 이상 플래그 |

> 임계값은 모두 "검토 필요" 상태로, 실데이터 분포 확인 후 최종 확정한다.

**통계 기반 이상 감지 (Statistical Outlier)**

| 방법 | 기준 | 설명 |
|---|---|---|
| Z-score | \|Z\| > 2.5 (검토 필요) | 상품 × 월 단위 표준화 후 이상치 판별 |
| IQR 방법 | Q3 + 1.5×IQR 초과 | 손해율 분포 상단 이상치 탐지 |

#### 3.5 집계 흐름

```
[원천 데이터]
  → 유효 계약 필터 (earned_premium > 0)
  → product_code × contract_ym 단위 집계
  → 손해율 산출 (파생 또는 원천)
  → 이동평균·변화율 파생 지표 산출
  → 임계값 비교 → 이상 플래그 생성
  → 상품군(product_group) 롤업
  → 채널(channel_code) 서브 집계
  → 최종 출력 테이블
```

| 테스트 항목 | 기준값 | 검증 방법 | 담당 |
|---|---|---|---|
| 손해율 산출 정확성 | incurred_loss / earned_premium 수식 일치 | 샘플 5건 수동 계산 대비 | 검토 필요 |
| 집계 단위 누락 여부 | product_code 전체 집계 완전성 100% | 원천 대비 집계 건수 비교 | 검토 필요 |

| 이상치 Z-score | 임계 이상 건 누락 없음 | 전체 분포 확인 | 검토 필요 |

## M-02 유지율 분석 모델

### 1. 목적

#### 1.1 분석 목적

청약월 기준 코호트별 MMP(월납보험료) 금액 기준 유지율을 집계하고, 주요 회차(4·7·13·25회차)별 이탈 패턴을 추적하여 채널·상품별 유지 성과를 비교한다. 유지율이 낮은 코호트·채널의 손해 지표(loss_Amt, riskP)를 동시 추적하여 유지 부진과 손해 간의 연계 리스크를 관리한다.

#### 1.2 모델 적용 범위

| 항목 | 내용 |
|---|---|
| 분석 단위 | 청약월(코호트) × 기준년월 매트릭스 |
| 유지율 산식 | 유지MMP / 모집MMP × 100 (금액 기준) |
| 주요 분석 관점 | 코호트별 MMP 유지율, 주요 회차 이탈 패턴, 채널·상품별 유지 성과, 손해 연계 분석 |
| 기대 효과 | 조기 이탈 회차 식별, 유지율 취약 채널·상품군 관리, 유지율-손해율 연계 감지 |

#### 1.3 활용 컬럼

| 컬럼명 | 업무적 의미 | 비고 |
|---|---|---|
| 기준년월 | 집계 기준 연월 (YYYYMM) | 분석 시점 기준 |
| 청약월 | 가입 코호트 기준 연월 (YYYYMM) | 코호트 기준 |
| 상품코드 | 상품 코드 | 상품별 유지율 분리 |
| 상품군 | 상품군 분류 | 상위 분류 |
| 상품그룹 | 상품 그룹 | 중간 분류 |
| 상품명 | 상품명 | 상품 식별용 |
| 플랜구분 | 플랜 구분 | 플랜별 유지율 분리 |
| 연령대 | 연령 구간 | 비식별 그룹 |
| TMRID | TM 담당자 ID | 비식별 사용 |
| center명 | 센터명 | 조직 단위 집계 |
| agency명 | 대리점명 | 채널 단위 집계 |
| 채널 | GA / CM / OB / 기타 | 채널별 분리 |
| 회차 | 납입 회차 | 경과 기간 기준 |
| 회차구분 | 주요회차(4·7·13·25) / 그외 | 조기 이탈 핵심 시점 |
| 모집MMP | 신계약 월납보험료 | 유지율 분모 기준 |
| 유지MMP | 유지 월납보험료 | 유지율 분자 기준 |
| 총지급보험금 | 총 지급 보험금 | 손해 연계 지표 |
| 월납보험료 | 월납 보험료 | 보조 기준 |
| riskP | 위험보험료 | M-01·M-02 공통 체계 |
| GWP | 보험료 | M-01·M-02 공통 체계 |
| loss_Amt | 보험금 | 손해율 연계 |

### 3. 변경/설계안

#### 3.1 모델 유형

코호트 분석 + MMP 기반 유지율 집계 (Cohort Analysis + MMP-based Retention Aggregation)

- 계약 단건 lapse_yn 기반 생존 분석(Kaplan-Meier) 구조를 폐기
- 집계된 MMP 금액(모집MMP / 유지MMP) 기준 코호트 유지율 산출 구조로 전면 재설계
- 주요 회차(회차구분) 기준 이탈 패턴 집중 분석 병행

#### 3.2 집계 기준

| 항목 | 기준 |
|---|---|
| 기준일 | 기준년월(YYYYMM) 기준 월말 마감 |
| 코호트 기준 | 청약월(YYYYMM) — 신계약 모집 시점 |
| 경과 기간 산출 | (기준년월 - 청약월) 월 차이 계산 |
| 집계 단위 | 청약월 × 기준년월 × 채널 × 상품코드 |
| 상위 집계 | 상품군 / 상품그룹 단위 롤업 집계 병행 |
| 조직 드릴다운 | agency명 → center명 단계적 집계 |
| 필터 조건 | 모집MMP > 0인 유효 코호트 기준 (이상 데이터 제외 조건은 실데이터 확인 후 확정) |

#### 3.3 핵심 지표 정의

**MMP 기반 유지율 지표**

| 지표명 | 산식 | 설명 |
|---|---|---|
| MMP 기준 유지율 | 유지MMP / 모집MMP × 100 | 핵심 KPI. 금액 기준 유지율 |
| 유지율 전월 대비 변화 | 당월 유지율 - 전월 유지율 | 유지율 추이 변화 감지 |
| 코호트 누적 유지율 | 기준년월 기준 청약월별 누적 유지율 | 코호트 경과 기간별 잔존율 |
| 이탈 MMP | 모집MMP - 유지MMP | 절대 금액 기준 이탈 규모 |

**주요 회차별 유지율 지표**

| 회차구분 | 경과 기간 의미 | 분석 목적 |
|---|---|---|
| 4회차 | 3개월 납입 완료 시점 | 초기 이탈 1차 확인 — 가입 후 최초 이탈 집중 구간 |
| 7회차 | 6개월 납입 완료 시점 | 초기 이탈 2차 확인 — 단기 유지 한계 구간 |
| 13회차 | 1년 납입 완료 시점 | 연간 유지율 기준 — 정책 효과 1년 단위 평가 |
| 25회차 | 2년 납입 완료 시점 | 장기 유지율 기준 — 2년 코호트 잔존 성과 |

> 회차구분 컬럼에서 '주요회차(4·7·13·25)'에 해당하는 레코드를 별도 서브셋으로 분리하여 집중 분석한다.

**채널·상품별 유지 성과 비교 지표**

| 지표명 | 집계 기준 | 설명 |
|---|---|---|
| 채널별 MMP 유지율 | 채널 × 청약월 × 기준년월 | GA/CM/OB 채널 간 유지 성과 비교 |
| 상품군별 유지율 | 상품군 × 청약월 코호트 | 상품군 단위 유지 성과 비교 |
| 채널 × 상품군 × 코호트 3-way 분석 | 채널 × 상품군 × 청약월 | 채널·상품군 조합별 유지 취약 구간 식별 |
| 플랜별 유지율 | 플랜구분 × 청약월 | 플랜 유형별 이탈 패턴 비교 |
| 연령대별 유지율 | 연령대 × 청약월 | 연령 구간별 이탈 패턴 확인 |
| 대리점별 유지율 | agency명 → center명 드릴다운 | 조직 단위 유지 성과 집계 |

**손해 연계 지표**

| 지표명 | 산식 | 설명 |
|---|---|---|
| 손해율 (riskP 기준) | loss_Amt / riskP × 100 | 위험보험료 대비 보험금 비율 |
| 총지급보험금 대비 모집MMP | 총지급보험금 / 모집MMP × 100 | 모집 규모 대비 지급 손해 비율 |
| 유지율 하위 코호트 손해율 | 유지율 하위 N% 코호트의 loss_Amt / riskP | 이탈 집중 코호트의 손해 수준 연계 확인 |
| 채널별 유지율-손해율 동시 추적 | 채널 × 청약월 기준 유지율 + 손해율 병렬 집계 | 유지 부진 채널의 손해 집중 여부 감지 |

#### 3.4 코호트 매트릭스 구조

**기본 매트릭스**

- 행(Row): 청약월 (YYYYMM) — 가입 코호트
- 열(Column): 기준년월 (YYYYMM) 또는 경과 회차 (4·7·13·25·36·...)
- 셀(Cell): 해당 코호트 × 기준 시점의 유지MMP / 모집MMP × 100 (%)

**매트릭스 예시 구조**

## M-03 심사데이터 분석 모델

### 1. 목적

#### 1.1 분석 목적

UW 심사통계 SQL 기반의 실제 확정 컬럼을 활용하여, 자동/인수심사 건수 추이, 알릴사항 고지 패턴, KCD 상병 집중도, 거절 사유 텍스트 분석, 채널·상품별 심사 분포를 집계·분석한다. 이를 통해 심사 품질 모니터링 및 인수 기준 고도화 근거를 제공한다.

#### 1.2 모델 적용 범위

| 항목 | 내용 |
|---|---|
| 분석 단위 | 자동심사월(YYYYMM) × 상품코드 × 채널 |
| 심사 유형 분류 | 자동심사 / 인수심사 (자동/인심사 컬럼 기반) |
| 주요 분석 관점 | 심사 유형 비율 추이, 알릴사항 고지 패턴, KCD 상병 집중도, 거절 패턴, 채널·상품별 분포, 피보험자 프로파일 |
| 기대 효과 | 심사 유형 효율 모니터링, 고지 항목 기반 인수 기준 고도화 근거 제공 |

### 3. 변경/설계안

#### 3.1 모델 유형

분포 집계 + 텍스트 빈도 분석 (Distribution Aggregation + Text Frequency Analysis)

- 자동/인심사 구분 기반 심사 유형 비율 통계 집계
- 알릴사항 항목별 빈도 및 KCD 코드 집중도 집계
- 거절메모·UW메모 개인식별 제거 후 키워드 빈도 분석

#### 3.2 집계 기준

| 항목 | 기준 |
|---|---|
| 기준일 | 자동심사월(YYYYMM) 기준 월말 마감 |
| 집계 단위 | 자동심사월 × 상품코드 × 채널 |
| 상위 집계 | 상품구분(Uw_product_G) 및 상품군(Uw_productGroup_G) 단위 롤업 집계 병행 |
| 채널 분리 | 채널 컬럼 기준 GA / CM / OB / 기타 서브 집계 |
| 필터 조건 | 유효 심사 건 기준 (취소·무효 제외 조건은 실데이터 제공 시 확정) |
| 식별 정보 처리 | 증권번호·청약번호는 COUNT 집계에만 사용. 텍스트 컬럼은 개인식별 표현 제거 후 활용 |

#### 3.3 핵심 지표 정의

**자동/인심사 비율 지표**

| 지표명 | 산식 | 설명 |
|---|---|---|
| 자동심사 비율 | 자동심사 건수 / 전체 심사 건수 × 100 | 자동 처리 효율 추이 모니터링 |
| 인수심사 비율 | 인수심사 건수 / 전체 심사 건수 × 100 | 수동 심사 집중 구간 식별 |
| 채널별 자동심사 비율 | 채널 × 자동/인심사 조합 건수 / 채널 전체 건수 × 100 | 채널별 처리 유형 패턴 비교 |
| 상품별 인수심사 집중도 | 상품코드별 인수심사 건수 / 상품코드별 전체 건수 × 100 | 수동 심사 집중 상품 식별 |
| 월별 유형 전환 추이 | 자동심사월 기준 자동/인심사 비율 월별 변화 | 심사 유형 비율 변화 감지 |

|---|---|---|

| 알릴사항2 항목별 빈도 | 알릴사항2 값별 건수 / 알릴사항2 입력 건수 × 100 | 2차 고지 패턴 분포 |
| 알릴사항3 항목별 빈도 | 알릴사항3 값별 건수 / 알릴사항3 입력 건수 × 100 | 3차 고지 패턴 분포 |
| 고지 항목 복수 입력 비율 | 알릴사항2 이상 입력 건수 / 전체 건수 × 100 | 복수 고지 집중 구간 확인 |
| 상위 고지 항목 랭킹 | 알릴사항1~3 통합 항목별 빈도 합산 Top-20 | 전체 고지 패턴 요약 |

**KCD 상병 집중도 지표**

| 채널 × KCD 조합 분포 | 채널별 상위 KCD 코드 빈도 분리 집계 | 채널별 고지 상병 패턴 비교 |

> 텍스트 분석 전 반드시 성명, 주민등록번호, 연락처 등 개인식별 가능 패턴을 마스킹·제거한 후 분석에 활용한다.

> 현재 M-03 심사통계 데이터와 M-01 손해율 데이터 간 직접 조인 키(증권번호 등)는 출력물에서 사용하지 않는다.

#### 3.4 5차원 위험 스코어링 설계

##### 3.4.1 차원별 구성 및 가중치

| 차원 | 전체 가중치 | 구성 지표 및 세부 가중치 |
|---|---|---|
| DIM1 근접사고위험 | 25% | 근접건수(30%) + 근접비율(30%) + 최소경과일역순(25%) + 7일이내건수(15%) |
| DIM2 유형집중위험 | 18% | 유형집중도(35%) + 단일유형100%(35점 고정) + 담보집중도(20%) + 다발피보험자(10%) |
| DIM3 부정사기위험 | 25% | 사기의심건수(30%) + 사기의심비율(20%) + 사고발생후가입(20%) + 가입전사고(10%) + 텍스트사기징후(10%) + 내용짧음(10%) |
| DIM4 고손해볼륨위험 | 17% | 사고건수(25%) + 보험금합(30%) + 건당보험금(20%) + 다발피보험자비율(15%) + 고위험직군(10%) |
| DIM5 이력행태위험 | 15% | 현재유의(30점 고정) + 이력유의(15점 고정) + 유의피보험자(15%) + 설계사본인청구(20%) + 해약비율(10%) + 면책비율(10%) |

> DIM2·DIM5의 고정 점수 항목(단일유형100%, 현재유의, 이력유의)은 조건 충족 시 지정 점수를 직접 가산한다.
> 나머지 항목은 백분위 기반 표준화(rank(pct=True) × 100) 후 세부 가중치 적용.
> 전체 가중치 합계: 25 + 18 + 25 + 17 + 15 = 100%.

##### 3.4.2 스코어 산출 로직

```
[1단계] 피처 산출
  - 15개 카테고리(A~O) 기준 취급자코드 단위 집계

[2단계] 차원별 표준화
  - 백분위 기반: rank(pct=True) × 100 (취급자 전체 모집단 기준)
  - 고정 점수 항목은 조건 충족 여부로 직접 점수 부여

[3단계] 차원 점수 산출
  - 각 차원 내 구성 지표 × 세부 가중치 합산 → 차원 점수 (0~100)

[4단계] 종합 점수 산출
  - risk_score = DIM1×0.25 + DIM2×0.18 + DIM3×0.25 + DIM4×0.17 + DIM5×0.15
  - 최종 범위: 0~100 (높을수록 고위험)

[5단계] 등급 분류
  - 절대값 기준 4단계 분류 적용
```

#### 3.5 등급 분류 체계 (절대값 기준)

| 등급 | 점수 기준 | 권고조치 |
|---|---|---|
| CRITICAL | 65점 이상 | 즉시 설계 전면 제어 + 심사강화 |
| HIGH | 48~64점 | 신계약 심사강화 + 모니터링 강화 |
| MED | 32~47점 | 정기 모니터링 + 이상 시 알림 |
| LOW | 32점 미만 | 정상 관리 |

> 등급 기준은 절대값 기반으로 고정하며, 분위수 기반 분류를 병행 검토하지 않는다.

#### 3.6 손해 증가 패턴 감지 (4개 분석축)

| 분석축 | 집계 단위 | 분석 내용 |
|---|---|---|
| 1. 사고유형별 월별 손해 추이 | 사고유형 × 월 | 최근 6개월 vs 이전 6개월 건수/보험금 증가율 비교 |
| 2. 대리점별 손해 증가 패턴 | 대리점코드 | 대리점코드 단위 보험금 증가율 상위 구간 분석 |

> 분석축 1의 비교 기간(최근 6개월 vs 이전 6개월)은 기준일 기준 rolling 적용.
